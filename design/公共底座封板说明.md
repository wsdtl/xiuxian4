# 公共底座封板说明

## 封板状态

```text
public-foundation.v7
```

`public-foundation.v7` 在 `v6` 的具名世界皮肤与角色名称契约上增加紧凑展示名、单行角色名规则和
等级派生境界，使聊天人物头无需称号统计或额外持久字段。

`public-foundation.v6` 在 `v5` 的治理基线上补齐具名世界皮肤、角色名称与发送者名称契约，并建立
正式内容层和具体游戏规则层。正式名录与双世界皮肤可以开始随组件逐项扩展。封板
表示公开契约、依赖方向和持久化边界已经稳定；具体玩法不能绕过这些边界。

## 封板范围

- `launch/`：应用生命周期、模块加载、通信驱动器、命令上下文和显式回复目标。
- `message/`：跨驱动消息语义、富文本结构、动作协议、渲染器和可扩展图标注册表。
- `game/core/`：账号、规则、战斗、资产、装配、经济、奖励、权益兑付、行动、周期、内容和持久化底座。
- `game/content/`：具体游戏的稳定名录、官方世界皮肤和统一内容装配。
- `game/rules/`：跨组件复用的具体游戏规则策略。
- `game/app.py`：数据库、内容、规则与应用服务装配。
- `game/cmd/`：命令与 HTTP 接口总路由，只负责接入和调用编排。
- `auto/`：与具体游戏无关的启动初始化模块。
- `组件测试/`：真实 QQ 协议现场探针，封板后继续保留。
- `test/`：公共底座自动化回归测试，永久保留。

## 依赖方向

```text
game/content         -> game.core.gameplay
game/rules           -> game.core + game.content
game/app.py          -> game.rules + game.content + game.core.persistence + launch
game/cmd/*/service   -> game.rules + game.app + message + launch
launch             -> message
组件测试           -> message + launch
message            -> Python 标准库
game.core          -> Python 标准库与自身领域
```

`launch` 和 `message` 不得导入 `game`。`game.core` 不得导入命令、消息协议、通信驱动器或测试组件。协议测试不得依赖游戏代码。

## 扩展规则

1. 新物品、技能、效果、角色、装备、武器和世界内容优先通过定义、标签、注册表与内容包扩展。
2. 新游戏图标通过 `message.register_icons(source, icons)` 注册，不修改公共图标表。
3. 新玩法编排调用公共事务和规则能力，不直接写核心聚合或数据库表。
4. 兑换码、小游戏结果、活动补偿和迁移资产必须先形成权益，再通过统一奖励结算兑付。
5. 只有公共底座确实无法表达一种新能力时才修改核心，并补齐跨领域回归测试。
6. 破坏公开契约、存储格式或事件语义时必须升级底座版本，不能增加旧路径兼容壳掩盖变化。
7. 品质由生成后的内在价值决定；内容包必须在启动期完成全部随机贡献的严格估值校验。
8. 武器随机属性遵守机制兼容约束，装备随机属性保持开放；上层不得偷偷加入定向洗词条旁路。
9. 队伍邀请使用社会请求表达同意，接受邀请与加入队伍必须在同一个数据库事务中提交。
10. 队伍只拥有成员关系；战斗、奖励、掉落和具体玩法状态继续由各自领域拥有。
11. 全服活动只保存一份状态；具体活动通过注册目录接入统一列表和热点通栏，不能为每个玩家复制活动通知。

## 不在封板范围

- 新游戏名称、世界设定、数值表和内容包。
- 玩家正式命令、主循环、教程和运营活动。
- 具体小游戏 HTTP API、鉴权和管理页面。
- 旧项目代码、旧数据库和历史数据迁移。

## 验证

封板要求以下验证同时通过：

```powershell
.venv\Scripts\python.exe -X utf8 -B test\core_architecture_test.py
.venv\Scripts\python.exe -X utf8 -B test\core_governance_test.py
.venv\Scripts\python.exe -X utf8 -B test\message_protocol_test.py

Get-ChildItem test\*_test.py | Sort-Object Name | ForEach-Object {
  .venv\Scripts\python.exe -X utf8 -B $_.FullName
  if ($LASTEXITCODE -ne 0) { exit $LASTEXITCODE }
}
```

真实服务还必须完成生命周期启动、QQ 驱动加载和 HTTPS 探测。自动化测试与 `组件测试/` 不因封板完成而删除。
