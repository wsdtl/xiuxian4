# 公共底座封板说明

## 封板状态

```text
public-foundation.v1
```

`public-foundation.v1` 是新游戏内容开始建设前的公共底座基线。封板表示公开契约、依赖方向和持久化边界已经稳定；后续可以增加具体内容，但不能为了单个玩法绕过这些边界。

## 封板范围

- `launch/`：应用生命周期、模块加载、通信驱动器、命令上下文和显式回复目标。
- `message/`：跨驱动消息语义、富文本结构、动作协议、渲染器和可扩展图标注册表。
- `game/core/`：账号、规则、战斗、资产、装配、经济、奖励、行动、周期、内容和持久化底座。
- `game/cmd/`：命令与 HTTP 接口总路由，只负责接入和调用编排。
- `auto/`：与具体游戏无关的启动初始化模块。
- `组件测试/`：真实 QQ 协议现场探针，封板后继续保留。
- `test/`：公共底座自动化回归测试，永久保留。

## 依赖方向

```text
未来游戏内容与编排 -> game.core
未来命令组件       -> game.core + message + launch
launch             -> message
组件测试           -> message + launch
message            -> Python 标准库
game.core          -> Python 标准库与自身领域
```

`launch` 和 `message` 不得导入 `game`。`game.core` 不得导入命令、消息协议、通信驱动器或测试组件。协议测试不得依赖游戏代码。

## 扩展规则

1. 新物品、技能、效果、角色、装备、武器和世界内容优先通过定义、标签、注册表与内容包扩展。
2. 新游戏图标通过 `message.register_icons(source, icons)` 注册，不修改公共图标表。
3. 新玩法编排调用公共事务和规则能力，不直接写核心聚合或数据库表。
4. 只有公共底座确实无法表达一种新能力时才修改核心，并补齐跨领域回归测试。
5. 破坏公开契约、存储格式或事件语义时必须升级底座版本，不能增加旧路径兼容壳掩盖变化。

## 不在封板范围

- 新游戏名称、世界设定、数值表和内容包。
- 玩家正式命令、主循环、教程和运营活动。
- 待定后台接口的业务 API、鉴权和管理页面。
- 旧项目代码、旧数据库和历史数据迁移。

## 验证

封板要求以下验证同时通过：

```powershell
.venv\Scripts\python.exe -X utf8 -B test\core_architecture_test.py
.venv\Scripts\python.exe -X utf8 -B test\message_protocol_test.py

Get-ChildItem test\*_test.py | Sort-Object Name | ForEach-Object {
  .venv\Scripts\python.exe -X utf8 -B $_.FullName
  if ($LASTEXITCODE -ne 0) { exit $LASTEXITCODE }
}
```

真实服务还必须完成生命周期启动、QQ 驱动加载和 HTTPS 探测。自动化测试与 `组件测试/` 不因封板完成而删除。
