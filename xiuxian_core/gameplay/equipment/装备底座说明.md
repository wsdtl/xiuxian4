# 装备底座说明

## 封板版本

```text
equipment.foundation.v1
```

装备负责辅助、转化和连接武器战斗循环。玩家同时可以激活六件装备。

## 唯一结构

```text
槽位 + 流派 + 品质
```

装备实例只保存装备定义和品质。装备定义已经固定槽位与流派，因此实例不重复保存这两个字段。

装备明确没有：

```text
等级
经验
强化
随机词条
宝石
孔位
附魔
技能书
套装计数奖励
```

## 流派与品质

流派决定机制方向，同一流派在六个槽位可以提供不同但互相连接的贡献。每件装备自身提供规则，
不要求穿满固定件数才启动玩法，因此同流派和混搭都能直接工作。

品质只选择该装备定义已经写明的贡献参数：

```text
普通：触发率 0.15
优秀：触发率 0.22
稀有：触发率 0.30
珍贵：触发率 0.40
```

低品质已经拥有完整机制，高品质只提高明确数值。品质参数作为内容配置保存，不在装备引擎中
使用隐藏公式。

## 角色贡献

`EquipmentContributionProvider` 合并流派标签、装备基础贡献和当前品质贡献，产生一份来源为具体
装备实例的 `CharacterContribution`。属性、Ability、Trigger 和伤害干预仍由 Gameplay 内核处理。

完全相同的装备仍可作为独立实例保存来源，但聊天输出可以按“槽位 + 流派 + 品质”聚合数量，
不要求玩家管理一串实例编号。

## 不属于本底座

- 具体流派、装备和品质数值。
- 装备等级与经验。
- 附魔、宝石、孔位、强化和随机词条。
- 掉落、交易、自动回收和展示文案。
- 实例铭刻由独立铭刻底座承接。
