# 账号与归属底座说明

## 封板版本

```text
account.foundation.v1
```

账号底座位于核心包 `xiuxian_core/account/`，与 Gameplay 平级。它负责把通信平台身份转换为稳定内部账号，
但不处理消息回复、角色数值、库存和经济。

## 身份与回复目标分离

`MessageContext.client_id` 和 `ReplyTarget` 只服务当前消息路由，不能成为永久玩家主键。

QQ 私聊和群聊可能分别使用：

```text
user_openid
member_openid
```

账号底座把它们转换为多个 `ExternalIdentity`，最终共同指向一个内部 `account_id`。

## 外部身份键

完整唯一键包含：

```text
provider_id
tenant_id
subject_kind
scope_id
external_id
```

QQ 示例：

```text
platform.qq + 机器人AppID + identity.qq_user + 空作用域 + user_openid
platform.qq + 机器人AppID + identity.qq_group_member + 群openid + member_openid
```

机器人 AppID 防止不同机器人应用的 OpenID 撞号；群作用域防止群成员身份被跨群误认。

## 自动归并

`IdentityEvidence` 只接受同一个已验证平台事件同时提供的一组身份。解析规则：

1. 所有身份都未绑定：创建内部账号并绑定全部身份。
2. 已绑定身份只指向一个账号：把缺少的身份补绑到该账号。
3. 所有身份已经指向同一账号：直接返回该账号。
4. 身份分别指向多个账号：保留原绑定并记录冲突，不覆盖、不猜测。

因此第一次群消息同时提供 `U123/M456` 后，两条记录都会指向同一账号。以后私聊只提供
`U123`，仍然可以解析到原账号；进入另一个群时，再通过同一事件中的 `U123` 自动补绑新群成员
身份。

昵称、头像、群名片和玩家自述不能作为自动归并证据。

## 防重放与并发约束

QQ 凭据 ID 使用：

```text
qq:机器人AppID:平台事件ID
```

同一凭据重复提交会返回第一次结果；相同凭据 ID 如果携带不同身份集合，会得到
`account.evidence_mismatch`。

当前纯底座使用不可变目录表达唯一关系。接入数据库时必须建立：

```text
UNIQUE(provider_id, tenant_id, subject_kind, scope_id, external_id)
UNIQUE(evidence_id)
UNIQUE(transaction_id)
```

并在唯一约束竞争后重新读取绑定，不能创建第二个账号。

## 账号状态

```text
active
suspended
closed
```

暂停账号保留全部资产但不能执行游戏业务；关闭账号不能重新启用。状态修改使用账号 revision 和
事务 ID，拒绝旧快照覆盖和重复执行。

解绑外部身份不会删除账号或资产，并且禁止解绑最后一个登录身份。

## 角色归属

角色永久字段明确为：

```python
CharacterState.account_id
```

不再使用含义模糊的 `owner_id`，也不在账号中复制角色列表。角色记录是账号归属的唯一事实。

通用 `PrincipalRef` 可以区分：

```text
principal.account
principal.character
principal.system
principal.business
```

供后续经济账户、系统托管和业务主体使用，避免只靠字符串前缀猜所有者类型。

## QQ 凭据桥

`build_qq_identity_evidence(...)` 接收 QQ 驱动已经完成签名校验后的字段，不导入 QQ 驱动代码。
当 `user_openid/member_openid` 同时存在时自动形成一组身份；只有兼容 `actor_openid` 时会生成
独立的 `identity.qq_actor_compat` 身份，不会假装它就是用户 OpenID。

## 结构化事实

```text
account.created
account.identity.bound
account.identity.unbound
account.identity.conflict
account.status.changed
```

日志事件只保存身份指纹，不写入原始 OpenID。持久化层需要使用可查询的 HMAC 或受保护字段保存
真实身份键，普通日志不得输出原值。

## 不属于本底座

- 角色创建数量策略。
- 管理员权限和游戏称号。
- 账号资料和玩家可见昵称。
- 数据库表与加密密钥实现。
- 经济余额、物品和装备。
- QQ 回复目标和主动推送地址。

状态、身份解绑和角色归属。
