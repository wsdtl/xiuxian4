# xiuxian4 核心边界说明

## 核心版本

```text
xiuxian-core.v1
```

`xiuxian_core/` 只收纳不依赖命令和通信平台的游戏地基：

```text
xiuxian_core/
├─ gameplay/       纯规则、内容目录和规则引擎
├─ account/        平台中立的账号与身份归属
├─ persistence/    数据库、快照、联合事务和 Outbox 适配器
└─ 核心边界说明.md
```

## 三层职责

`gameplay/` 只计算规则结果。它不能读取数据库、机器时间、QQ 事件或消息上下文，也不能导入
`persistence`、`launch`、`message` 和 `components`。

`account/` 只把平台身份映射为稳定内部账号。它不知道 QQ 原生协议、Gameplay 状态和数据库实现，
同样不能导入其他核心层。

`persistence/` 是核心内部唯一允许认识领域快照和 SQLite 的适配层。它可以依赖 `gameplay` 和
`account`，但不能依赖 `launch`、`message`、`components` 或具体玩法命令。

## 核心之外

```text
launch/       应用生命周期和通信驱动器
message/      跨驱动消息协议
components/   命令入口和具体玩法编排
xiuxian_game/ 具体世界内容与跨领域玩法编排
static/       网页资源
design/       游戏设计文档
test/         测试与架构巡检
```

这些目录可以在自己的职责内调用核心，但核心不能反向调用它们。QQ 文案、按钮、命令参数、HTTP
路由和定时 worker 启动方式都不属于核心。

行动领域只维护行动槽、冻结快照和生命周期。具体探险结果由 `xiuxian_game/` 计算，资产变化由奖励
结算提交，周期系统只提供时间窗口；三者不能互相复制职责。

## 导入规则

外部代码必须使用完整产品命名空间：

```python
from xiuxian_core.gameplay.inventory import InventoryEngine
from xiuxian_core.account import AccountIdentityEngine
from xiuxian_core.persistence import SqliteDatabase
```

禁止重新建立顶层 `gameplay`、`account`、`persistence` 兼容包。核心内部优先使用相对导入；根
`xiuxian_core.__init__` 只公开整体版本和层清单，不聚合所有领域类型。

## 验证

```powershell
.venv\Scripts\python.exe -X utf8 -B test\core_architecture_test.py
```

架构测试使用 Python AST 检查导入关系，并验证旧顶层包不存在、核心根入口保持精简。
