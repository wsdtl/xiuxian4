# 首个世界说明

`xiuxian_game/` 是具体修仙内容与玩法编排层。它可以依赖 `xiuxian_core`，但不能依赖命令、消息协议、
QQ 驱动器或 HTTP 路由。

## 当前闭环

```text
开始修仙
-> 原子创建账号归属、角色、纳戒、账本、青竹剑、装配和领取状态
-> 行动系统创建并完成山门试炼，冻结角色、装配、规则版本、内容指纹和随机种子
-> 叩山一击通过核心 Ability 击破木傀
-> 领取 25 灵石、20 经验和清露草 x2
-> 清露草通过统一物品使用事务恢复 10 精神
-> 装备青竹剑
-> 状态与纳戒展示持久结果
```

QQ 群聊事件同时出现 `user_openid/member_openid` 时会自动绑定到同一内部账号。玩家创建、行动状态、
待领取结果、奖励事务和装备状态都保存到 SQLite；服务重启后继续读取。重复行动复用当前待领取结果，
重复领取不会再次发放奖励。山门试炼不再拥有专用持久化状态，后续探险和休息会复用同一行动生命周期。

## 世界内容

首个内容包包含：

- 云游散修角色模板和五级修行境界。
- 灵石、凡品、青竹剑和六件云游装备定义。
- 清露草物资、服用 Ability 和“目标精神未满”条件；服用后恢复 10 精神。
- 叩山一击 Ability、劲力伤害类型和山门木傀。
- 上海时区每日 04:00 周期。
- 占用主行动槽的山门试炼定义。
- 雾竹林探险与吐纳调息行动定义；探险消耗精神并由统一奖励系统结算。
- 第一套完整世界皮肤。

这组内容只用于验证正式闭环，不代表最终数值规模或完整玩法数量。

当前 `GameApplication` 已提供协议无关的可用物品查询和自身使用接口，支持数据库防重、重启重放、
满精神不消耗和其他账号目标拒绝。QQ 命令与展示组件尚未接入。
