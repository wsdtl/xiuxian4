# 队伍正式业务

队伍是轻量、可长期保留的玩家成员关系。队伍领域只建设关系底座，不接入探险、灾厄、奖励或掉落；
已经落地的组队挑战只读取队伍状态，不把战斗规则写回队伍领域。

## 状态边界

- 队伍核心只保存队长、成员、站位、准备状态和生命周期。
- 一名角色同时只能属于一支活跃队伍。
- 队伍容量为 3 人；伙伴不占队伍成员名额。
- 成员或站位变化会清空全员准备状态。
- 队伍不拥有共享背包、货币、装备、伙伴、等级、经验、增益或奖励。
- 邀请使用社会请求底座；接受邀请与加入队伍由核心联合事务原子提交。

## 当前命令

- `组队`：查看当前队伍、成员站位、准备状态和收到的邀请。
- `创建队伍`：创建三人队伍并成为队长。
- `邀请组队 玩家`：队长邀请指定玩家，邀请十分钟内有效。
- `接受组队 请求编号`：接受属于当前角色的邀请。
- `拒绝组队 请求编号`：拒绝属于当前角色的邀请。
- `退出队伍`：普通成员离队；队长需要先转让队长或解散队伍。
- `请离队伍 玩家`：队长移除成员。
- `转让队长 玩家`：队长把队长身份交给当前成员。
- `解散队伍`：队长解散队伍。
- `准备`、`取消准备`：修改自己的准备状态，供当前组队挑战使用。

所有写操作都通过队伍 revision 校验，按钮过期或多人同时操作时不会覆盖新状态。

队伍本身不绑定某个真实世界。组队挑战读取队伍快照后，由独立的组队挑战领域建立临时战斗投影；
队员的世界身份、地点存在体和跃迁状态不写入或改写。
