# 组队挑战

组队挑战是队伍关系之上的独立玩法，不把组队域改造成战斗域。

- `PartyFeature` 只负责队伍成员、队长、站位和准备状态。
- `PartyBattleFeature` 保存当前锁定的组队首领、准备指纹、尝试次数和每日奖励状态；队伍准备标记与挑战
  指纹由它在同一个工作单元提交，命令层不能分两次写入。
- 首领从各世界自己的组队首领池中抽取，个人探险首领不进入这个池子。
- 战斗复用公共战斗核心；三名玩家最多各带当前配装绑定的伙伴进入同一场战斗。
- 失败保留同一个首领供重试，清空准备指纹并结算真实资源损失。
- 胜利后每名玩家独立抽取自己的奖励；每日前两次胜利正常发放，之后只能助战不发成长和物品奖励。
- 玩家奖励、资源变化、挑战状态和完整战报在同一个 SQLite 工作单元中提交。
- 完整战报通过统一战报服务公开保存七天，QQ 只返回摘要和战报编号。

命令入口仍归属于 `game/cmd/组队`：`组队挑战`、`选择组队挑战 等级`、`开始组队挑战`。
