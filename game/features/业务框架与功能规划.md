# 业务框架与功能规划

## 目标

本文件锁定“公共底座之上如何增加正式玩法”。它不是玩法介绍，而是开发约束：新功能必须先确定业务
归属、状态所有权和事务边界，再增加命令。测试通过不能替代边界验收。

## 固定分层

1. `game/core`：协议和题材无关的规则积木、状态机、事务引擎与持久化实现。不得知道具体玩法名称。
2. `game/content`：稳定名录、概率、价格、周期和世界皮肤。不得读取数据库或处理玩家请求。
3. `game/rules/<domain>`：具体游戏的纯规则。输入相同就必须得到相同结果，不接触数据库、QQ 和机器时间。
4. `game/features/<domain>`：正式业务用例与联合事务。只接收组合根注入的数据库端口、仓储和领域引擎。
5. `game/cmd/<中文组件>`：命令注册、参数、按钮和最终展示。不得直接读写快照。
6. `game/app.py`：组合根和稳定门面。允许装配、异常映射和调用转发，禁止直接开启业务工作单元。

依赖只能从外向内。两个 feature 不直接导入彼此；需要组合时，由 `game/app.py` 注入共享端口，或者新增
职责明确的协调 feature。不能把两个业务状态合成一个万能聚合。

## 已落地业务

实际台账定义在 `game/features/catalog.py`，治理测试会核对目录、命令组件和定时任务。

| 业务 | 主要职责 | 玩家入口 |
| --- | --- | --- |
| player | 身份到角色、角色总览、个人设置、提醒与活动读模型 | 角色、提醒、活动 |
| dimension_shift | 跃迁条件、凭证扣除与界相更新 | 跃迁 |
| exploration | 持续探险、战斗、掉落与奖励 | 探险 |
| rest | 行动占用与按时恢复 | 休息 |
| sparring | 邀请、真实配装切磋与战报 | 切磋 |
| dimensional_disaster | 全服灾厄、贡献、周期和唯一遗羽 | 多次元灾厄 |
| economy | 回收、二手交易、估价与税金 | 回收、二手 |
| lottery | 购票、开奖、退票和中奖入账 | 彩票 |
| draw | 扣签、保底和混合奖励 | 抽奖 |
| special_items | 带长期状态的特殊物品效果 | 物品 |
| battle_report | 所有战斗模式共用的公开战报 | 战报 |

装配、普通物品使用、铭刻和通知收件箱目前由稳定核心服务直接承接；它们不是缺失 feature，也不能为了
目录整齐再包一层空业务。只有出现跨领域规则或联合事务时，才升级为正式 feature。

## 新业务准入

新增业务前必须回答以下问题，并写入该业务的中文说明：

1. 业务唯一职责是什么，是否已经属于现有 feature？
2. 使用哪些内容 ID 和纯规则，平衡数字放在哪个 `content` 领域？
3. 读取和写入哪些聚合，哪个业务拥有写入口？
4. 哪些变化必须在同一工作单元提交，失败时回滚什么？
5. 是否需要幂等 ID、快照 codec、定时任务、公开战报或通知？
6. 命令组件只负责哪些参数、按钮和展示？
7. 最小闭环测试如何覆盖成功、重复、并发、资源不足和数据库重启？

未回答完不得创建命令目录。不得先加万能 `metadata`、大量可空字段或临时数据库表再寻找用途。

## 目录模板

正式业务按需要使用以下文件，不创建空占位：

```text
game/features/<domain>/
    __init__.py
    models.py       # 应用请求、结果和存储键
    service.py      # 唯一业务入口与工作单元
    codec.py        # 仅在拥有持久化业务状态时出现
    battle.py       # 仅在拥有独立战斗投影时出现
    说明.md
```

对应纯计算进入 `game/rules/<domain>`；概率、价格和正式名录进入 `game/content/catalog/<domain>`；玩家入口
进入 `game/cmd/<中文组件>`。`package.py` 只组装内容，`app.py` 只组装服务。

## 后续能力槽位

以下能力只有在玩法方案审核后才建立 feature，不提前创建目录或状态：

- 队伍玩法：公共 party 底座已经存在，但尚无正式组队收益循环。
- 境界突破：当前只有等级映射展示，尚无独立突破消耗与失败规则。
- 灵宠：可复用物品、成长、配装和战斗贡献接口，但必须先确定归属与持久化状态。
- 新全服事件：优先复用活动、周期、奖励和战报能力，不复制多次元灾厄状态。

这些是可扩展位置，不是待办承诺。具体玩法没有审核前，不进入数据库和命令注册。
