# 装配组件

装配组件负责当前七个槽位和六套配装的玩家交互。它不提供通用武库或物品详情查询；武库和物品详情
仍由对应的物品组件负责。

```text
装配
装备 物品编号
卸下 槽位
配装
配装 0-5
```

装备命令从物品定义自动取得唯一槽位。替换、卸下和配装切换统一调用 `PersistedLoadoutService`，库存位置
与装配状态在同一个 SQLite 工作单元中提交。当前配装发生装备或卸下后自动保存，不提供额外保存命令。

## 构筑与世界边界

- 一个武器槽和六个装备槽属于角色的持久构筑；六套配装分别保存引用，当前使用哪套只改变后续行动读取的构筑。
- 同一个武器或装备实例只能被一套配装引用，切换配装不会复制实例、刷新属性或改变物品编号。
- 武器和装备的品质、等级、随机属性、套装身份、战斗效果和铭刻数据归实例所有，不由世界皮肤重算。
- 装配页面显示当前世界的槽位名和装备名；世界皮肤只提供展示投影，跃迁不会改变装备实例、配装引用或战斗数值。
- 伙伴绑定属于当前配装的一个独立引用，装配组件只保存或读取该引用，不拥有伙伴实例和伙伴成长。
