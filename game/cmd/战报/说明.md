# 战报

- 所有战斗模式共用 `/battle/{share_id}`。
- 页面公开可分享，不读取 QQ 身份，也不检查战报所有者。
- QQ 只展示结算摘要和完整战报链接。
- 完整行动明细保留 7 天，随后只显示摘要；30 天后整份战报删除并返回 404。
- 页面按核心转场展示战斗建立、每次行动、外部阶段、加入和退出；每项都显示决策、请求目标、实际目标、完整事件链以及转移前后状态。
- 每个状态帧展示战斗状态、行动顺序、失活实体、胜方、行动进度，并区分正面、负面和其他效果，显示层数、剩余回合、属性、资源、冷却、触发器、拦截器与目标限制。
- 后端把结构化事实翻译成正文、标签、筛选项、状态差异和完整展示节点，不参与战斗和奖励结算。
- `/battle/{share_id}/data` 返回版本化公共战报文档，页面只按节点结构渲染，不理解任何战斗语义。
- 事件中文语义统一注册在 `game/features/battle_report/presentation.py`；新增普通内容不改网页，新增事件语义只补后端注册。
- 未注册事件直接视为协议缺口并使覆盖测试失败，不提供猜测性正文或浏览器兜底。
- 浏览器页面、样式和脚本位于 `static/battle-report/`，不得反向依赖战斗规则或识别具体武器、装备与特效。
- `jobs.py` 每天触发一次保留期清理，实际删除规则由 `BattleReportService` 执行。
