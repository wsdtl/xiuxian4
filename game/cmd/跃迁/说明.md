# 跃迁组件

```text
跃迁
跃迁 世界名称
```

角色创建时会从全部已开放世界中做一次可重放的随机降临。`跃迁` 展示界门当前连接和其他可登录世界；
指定世界后，界门封存当前构筑并在目标世界重新构筑化身。技术上原子更新
`CharacterWorldState.world_id` 并把角色存在体迁移到目标世界独立空间，不复制角色、资产、构筑或规则状态。
目标世界存在相同地点展示身份时，映射到目标世界自己的锚点与坐标；否则返回目标世界出生点。

跃迁属于即时操作，每次成功登录另一世界会从纳戒自动消耗一枚跃迁凭证。查看界门、目标无效、原地跃迁、
主要行动占用或凭证不足都不会消耗。相同目标保持幂等，不增加状态版本。成功后的业务正文和统一人物头
都立即使用新世界皮肤。

玩家可见命令只接受稳定 `world_id` 或世界名称。组件通过 `WorldViewCatalog` 解析，不能手写世界分支。
`skin_id` 只负责名称、说明和历史战报还原，不能决定位置、地点功能、属性、掉落、价值或战斗规则。
