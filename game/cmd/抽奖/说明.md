# 抽奖组件

本组件只负责命令、按钮、共用演出和结果展示。抽奖概率与奖项来自正式内容层，扣签、保底、库存、
金币和记录由 `game/features/draw` 在同一个 SQLite 事务中协调。

## 命令

- `抽奖`：消耗一张抽奖签。
- `十连抽奖`：一次消耗十张抽奖签，按单抽顺序推进全部保底槽。
- `抽奖奖池`：查看档位概率、当前持有量和保底进度。
- `抽奖记录`：查看最近十次抽奖结算。

所有世界共用 `static/draw` 下的五个 GIF。文件缺失时只隐藏演出，不影响已经完成的抽奖结算。

正式奖池为常规 `77%`、珍稀 `20%`、特殊物品 `2%`、破境凭证 `1%`。珍稀档使用原有十抽档位
保底；破境凭证使用独立五十抽保底，自然获得对应凭证会立即清零。保底触发时不替换本次普通结果，
而是额外发放一枚当前世界显示的破境凭证。
