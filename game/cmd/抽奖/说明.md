# 抽奖组件

战斗偶尔留下尚未决定最终形态的余响，界门会把其中一次稳定可能封存在抽奖签中。使用抽奖签不是向某个
机构兑奖，而是让界门将这次未定结果显化，并通过统一奖励结算写入角色资产。连续显化积累的保底计数
代表结果稳定度，不是归航公约或其他机构临时追加奖励。

本组件只负责命令、按钮、共用演出和结果展示。抽奖概率与奖项来自正式内容层，扣签、保底、库存、
金币和记录由 `game/features/draw` 在同一个 SQLite 事务中协调。

## 命令

- `抽奖`：消耗一张抽奖签，显化一次未定结果。
- `十连抽奖`：一次消耗十张抽奖签，按单抽顺序显化并推进全部保底槽。
- `抽奖奖池`：查看档位概率、当前持有量和保底进度。
- `抽奖记录`：查看最近十次抽奖结算。

所有世界共用 `static/draw` 下的五个 GIF，因为显化发生在界门档案层。文件缺失时只隐藏演出，不影响
已经完成的抽奖结算；最终奖励名称仍按玩家当前世界构筑。

正式奖池为常规 `77%`、珍稀 `20%`、特殊物品 `2%`、破境凭证 `1%`。珍稀档使用原有十抽档位
保底；破境凭证使用独立五十抽保底，自然获得对应凭证会立即清零。保底触发时不替换本次普通结果，
而是额外发放一枚当前世界显示的破境凭证。
