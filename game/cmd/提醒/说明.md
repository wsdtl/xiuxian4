# 提醒组件

## 通栏入口

统一回复装饰器只显示提醒摘要：

```text
📌 提醒: 3 项待领取 | 2 条未读通知
```

`提醒/__init__.py` 通过 `GameCommand.handler(intent_ids=...)` 把两个稳定回复意图和正式命令原子绑定。
QQ Markdown 会把摘要渲染成无边框内联命令链接；不支持内联命令的驱动器降级为普通文本。

```text
notifications
pending_actions
```

这两个英文入口只供通栏意图调用，不进入玩家公开命令列表。

## 读取边界

两个命令都只读取当前账号的有效未读通知和当前角色的已完成行动。打开列表不会自动标记通知已读，
也不会领取行动结果。每条通知提供内部 `notification_read` 意图；服务会重新校验通知接收账号和 revision
后标记单条已读。具体通知动作必须单独注册稳定意图；未注册的动作不生成链接。

通知领域数据只保存动作意图和结构化参数，不保存 QQ 链接或聊天命令。命令映射只存在于
`game/cmd/reply_intents.py` 和对应正式组件。
