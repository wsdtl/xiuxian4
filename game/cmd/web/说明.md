# Web 游戏台组件

`web` 返回受密码保护的网页入口。登录后，网页以固定技术身份
`local.message_console` 使用本地驱动器执行正式游戏命令；网页展示身份为“归航公约维护员”，
实际角色名为符合全局命名规则的“归航维护员”。首次登录通过正常的 `创建角色` 命令自动建档，
不绕过账号、角色初始化或访问守卫。

## 配置

在 `.env` 同时设置：

```env
WEB_CONSOLE_USERNAME=
WEB_CONSOLE_PASSWORD=
```

任一项为空时，`/game-console` 返回未配置状态。会话仅保存在进程内，十二小时过期；写请求必须同时
通过 HttpOnly Cookie、CSRF 令牌和同源校验。登录失败按来源限速。

## 消息流水

- `launch/message_events.py` 只发布协议中立的收发事实，不导入游戏组件。
- QQ 和本地驱动器发布完整正文、图片引用、按钮、无边框按钮和命令链接语义。
- 本组件订阅全部事实，以 SSE 实时推送，并通过消息编号回查交互，网页不能伪造按钮数据。
- 预输入动作只填写输入框；发送、回调和链接保持各自语义。
- 只有 Web 游戏台会为未命中命令补充“命令未识别”和帮助入口；QQ 与公共本地驱动器仍保持原有静默边界。

消息使用 `MESSAGE_CONSOLE_DATABASE_PATH` 指定的独立数据库，不进入 `DATABASE_PATH` 指定的正式
游戏数据库。SQLite 实现位于
`game/core/persistence/message_flows.py`，由 `game/app.py` 组装，组件不直接执行 SQL。数据库最多
保留 3000 条或 48 小时，内存保留 500 条，首屏读取 100 条，每次向上读取 50 条，页面最多保留
300 条 DOM 节点。图片只保存受控引用或临时文件，不把二进制写入数据库。

## 页面

页面资源位于 `static/game-console/`。布局使用完整可视高度和 `visualViewport`，兼容手机软键盘；
正文 HTML 由服务端白名单投影生成，外链只允许 HTTP、HTTPS 或站内绝对路径。
