# 角色组件

## 创建角色

```text
创建角色
创建角色 名称
```

显式名称优先；未提供名称时使用当前消息携带的平台昵称。两者都为空时返回填写提示。一个账号只能创建
一个角色，重复发送不会产生第二份角色、资产、货币或世界存在体。

命令只通过公共 `MessageHandler` 注册一次。QQ 与本地驱动器分别把已验证事件转换成
`MessageIdentity`，组件不读取 `QqMessageEvent`、`LocalCommandEvent`，也不能只拿 `client_id` 猜测
账号归属。本地驱动器使用独立的 `platform.local` 身份命名空间，只用于测试和离线调试。

命令入口只接收 `message` 参数。当前用户身份、平台昵称和回复目标由组件服务从公共
`MessageContext` 读取，不要求每条命令重复声明框架参数。

组件不创建数据库连接、不装配内容，也不直接处理持久化异常。`game.app` 返回稳定创建结果，
组件只构造公共 `DocumentMessage` 并交给当前驱动器发送。
