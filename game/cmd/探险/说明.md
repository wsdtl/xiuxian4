# 探险组件

本组件只负责 `探险`、`前往`、`开始探险`、`停止探险` 和 `探险总结` 的命令注册、参数解析与消息展示。
探险始终发生在角色当前登录的真实世界和当前位置；相同功能的区域在不同世界拥有独立坐标、名称和展示投影。
区域生态、遭遇生成和战斗属于 `game/rules/exploration`；跨角色、库存、掉落、奖励和世界快照的原子提交
由 `game/features/exploration` 负责。

探险开始后每十分钟形成一个批次。后台调度器会主动结算；玩家查询或停止时也会补算已经到期的批次，
因此重启不会吞掉已经经过的时间。战败会保留此前收益、把血气保持为零并停止探险；首版武库没有容量
上限，所以“容量已满停止”接口已经保留，但要等未来真正有限的战利品背包接入后才会触发。

批次状态保存敌人实例和结构化奖励引用，不保存世界皮肤翻译后的名称。探险页、最近遭遇、奖励、自动
用药和回收结果都在查询时按角色当前登录世界构筑展示；跃迁不会改变任何已经结算的掉落和数值事实。
跃迁不会把正在进行的探险伪装成另一世界的旧记录；探险状态仍按稳定世界和区域事实结算，展示名称只在
查询时重新投影。
最近一批获得的武器和装备可直接进入对应的 `查看 编号` 详情，再使用详情中的装配或回收入口处理。

`jobs.py` 每分钟触发一次到期批次发现；它只调用 `ExplorationFeature.settle_all_due()`，不处理战斗、
掉落或奖励规则。
