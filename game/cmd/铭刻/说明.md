# 铭刻组件

铭刻组件只负责铭刻预览、确认、原名展示偏好和消息排版。铭刻之羽消费与目标改名统一调用
`PersistedInscriptionService`，组件不能分别修改库存和实例数据。

在《万象行纪》的背景中，铭刻之羽是跨界灾厄结局凝成的唯一恒定记录。铭刻会把玩家选择的名字写入
跨世界档案，因此铭刻名高于任何世界的公共构筑名称，跃迁后仍然保持不变。

公开命令：

```text
铭刻
铭刻 羽毛编号 目标编号 新名称
铭刻能力 羽毛编号 武器编号 能力序号 新名称
铭刻原名
铭刻原名 开启
铭刻原名 关闭
```

铭刻首先生成预览，确认按钮通过内部命令提交。确认时重新读取库存并校验所有权、永久编号、目标类型与
revision。铭刻名称限制继续由铭刻底座统一执行。

## 名称与构筑边界

- 铭刻目标只有武器、装备和武器能力的展示名称，不改变武器或装备的品质、等级、随机属性、套装效果、
  攻击特效、能力机制或数值。
- 默认名称由当前世界的公共内容投影生成；铭刻名是跨世界档案中的玩家私有覆盖名，优先级高于世界公共名称。
- `铭刻原名关闭`只显示铭刻名；`铭刻原名开启`显示“铭刻名（当前世界完整原名）”。跃迁只改变括号内的公共原名。
- 武器、装备、能力和铭刻之羽都保留稳定独立编号。铭刻不会复制装备、改变装配槽位，也不会把目标转移到其他配装。
- 铭刻成功后消耗一枚铭刻之羽；原名展示偏好是角色设置，不属于武器、装备或世界状态。
