# 正式内容层

`game/content/` 是具体游戏的次核心内容层。它依赖 `game.core.gameplay` 的公共定义，但不能依赖
命令、消息协议、通信驱动器、账号实现或数据库。

```text
game/content/
├─ presentation/    跨世界的武器装备完整名称与评分投影
├─ catalog/
│  ├─ activity/     全服活动展示窗口与通栏数量
│  ├─ character/    人物定义、初始资产、身份与恢复参数
│  ├─ combat/       战斗定义、公共数值与价值映射
│  ├─ companion/    伙伴物种、世界秘境与品质资质参数
│  ├─ disaster/     多次元灾厄来源、固定叙事、周期与内容审计
│  ├─ enemy/        敌人身份、行为、威胁与掉落表
│  ├─ exploration/  探险区域、遭遇权重与掉落倾向
│  ├─ item/         消耗品等物品定义
│  ├─ weapon/       武器定义、蓝图、机制与平衡审计
│  ├─ equipment/    装备定义、蓝图、随机属性与平衡审计
│  ├─ world/        世界空间与地点定义
│  ├─ foundation.py 跨领域稳定基础定义
│  └─ package.py    汇总各领域名录
├─ world_skins/
│  ├─ cultivation/  太玄界，按人物、战斗、物品、武器和世界拆分
│  ├─ magic/        魔法世界，保持同样的领域结构
│  ├─ validation.py 世界皮肤共享的展示完整性校验
│  └─ package.py    汇总全部官方世界皮肤
└─ official.py      统一装配名录并缓存角色级世界视图
```

`game/core/gameplay/content/skins.py` 只定义 `SkinEntry`、`SkinPack`、`SkinCatalog` 和
`SkinProjector` 等协议中立契约；`game/content/world_skins/` 只提供当前游戏的具体展示数据。两层不能
混放，也不能把具体世界名称写回核心契约。

在《万象行纪》的正式背景中，每个世界都拥有独立文明、历史和物质形态。世界皮肤是界门根据目标世界
法则重构通用档案的技术实现，不表示多个世界只是同一地点的视觉换色。共享地点 ID 表示界门锚址能够
稳定映射到各世界落点，共享资产 ID 表示跨世界档案中的所有权和规则事实保持一致。

后续新增材料、消耗品、武器或装备时，先进入 `catalog/` 下对应领域包，再由 `package.py` 汇总；
随后在每个 `world_skins/<世界>/` 的同类模块补齐名称、说明和图标。禁止把全部定义继续堆进
`foundation.py`、单个世界文件或 `package.py`。同一领域一旦同时出现定义、蓝图、机制、属性、价值或
平衡审计等多种职责，必须在领域包内拆分模块，不得重新摊回 `catalog/` 根目录。根 `catalog` 入口只
导出稳定 ID、内容包和通用查询函数，内部编译器与审计器必须从具体领域子模块显式导入。

## 当前内容

```text
content.catalog.base@3.16.0
content.world_skins.official@3.13.0
skin.cultivation@20  太玄界
skin.magic@19        魔法世界
```

以上只记录当前已经实现的官方世界包，不声明诸界总数，也不限制未来世界题材与规模。

名录包已经登记主货币、五档品质、公共角色属性、默认人物、一百级成长表、基础攻击、六种药物、固定
新手剑、主世界和原点锚址。主货币在太玄界显示为“灵石”，在魔法世界显示为“魔晶”；同一个原点
锚址分别落在“太玄仙城”和“星辉王城”，两者是不同世界的独立城市。正式组件需要新内容时，先在名录中增加稳定 ID 和规则定义，
再同时为两套世界皮肤补齐名称、说明、图标和命令别名。

角色内容另外集中保存账号角色数、名称显示宽度、初始货币、背包容量、初始药品、六套配装、自动用药
阈值、休息恢复参数和唯一人物等级上限。活动内容集中保存热点展示窗口和通栏数量。规则层只读取这些
内容并执行校验、恢复或随机判定，不得重新声明相同数字。

多次元灾厄由每套启用世界皮肤贡献独立素材，也可以在未来来自世界碰撞、跨界文明或规则异常；正式名称、
故事、技能身份和遗羽文案进入界海公共灾厄名录，不经过角色当前世界皮肤投影。首批太玄界与魔法世界各贡献十只灾厄，其中七只取自公共领域
神话、传说或文献，三只为原创。该 `70%` 是内容审计软线，不是阻止服务器启动的硬断言。

“概率放在内容层”指概率值、权重表和可调阈值归内容所有；“概率判定放在规则层”指抽样、过滤、
互斥、回滚和确定性随机流程归规则所有。例如装备套装印记的 `25%` 在 `catalog/equipment`，规则生成器
只读取该值并执行一次判定。运行保护不是平衡内容：探险最多追赶多少批、调度器单次最多扫描多少会话，
由 `features` 保存并命名为操作上限。

特殊物品抽取统一使用内容包的 `draw_pools` 入口。每个池必须绑定唯一抽取签、版本化掉落表和可信
奖励身份集合，不能包含空奖或把自己的签作为奖励。抽取签的战斗掉率、正式特殊物品名录和保底阈值在对应
内容领域定义；命令与玩法层只能消费已冻结池，不能临时拼装奖励或权重。

首版抽奖在特殊物品名录为空时使用常规 `80%`、珍稀 `20%`；正式特殊物品出现后自动变为常规
`78%`、珍稀 `20%`、特殊 `2%`。常规和珍稀档内，金币奖项权重都为 `40%`：常规金币 `20`，
珍稀金币 `100`，其余权重由对应档位恢复药平分。金币是互斥奖项，不与药品叠加；当前单抽金币期望约
`14.4`，只补充零散流动性，不替代探险战利品和装备回收收入。

品质按规则排序固定为五档。太玄界由低到高显示为“黄、玄、地、天、圣”，魔法世界显示为
“普通、精良、稀有、史诗、传说”。品质属于武器和装备实例，不附加到固定消耗品。

基础药物分为小、中、大三档，分别恢复对应最大资源的 `12%`、`25%`、`50%`。修仙界显示为小还丹、
中还丹、大还丹和小回灵丹、中回灵丹、大回灵丹；魔法世界投影为对应的生命药剂和魔力药剂。角色创世
仍只发放两种小药，中药和大药由后续玩法产出。

纳戒物品固定分为三条互斥规则：恢复药是带 `item.medicine` 的可堆叠消耗品；特殊物品是带
`item.special` 的可堆叠消耗品，并且必须通过 `item_component.use_*` 类型化组件声明用途；铭刻之羽是
带 `item.inscription_medium` 的独立实例。恢复药和特殊物品带 `storage.special` 进入纳戒；铭刻之羽带
`storage.inscription` 进入铭刻系统独立保管区。只有恢复药会被自动用药识别，
铭刻之羽只能由铭刻联合事务消费。

首批伙伴内容包含两个世界各八种独立物种。物种名称、来源世界、角色实例和捕获图鉴属于跨世界事实，
不会随玩家跃迁改名；“伙伴、秘境、出战、放生”等公共操作词由当前世界投影。万灵引属于纳戒中的特殊
物品，使用时按当前世界开启对应秘境并一次固定三条不重复踪迹。伙伴品质使用公共五档品质，四项资质
预算、捕获权重、名册上限和秘境时长全部归 `catalog/companion`，随机选择与容量规则归
`rules/companion`。

正式铭刻之羽稳定 ID 为 `item.inscription.feather`。每枚实例仍必须携带独立标题和故事，不能堆叠或
复刻。铭刻后的私名高于世界皮肤完整原名；世界皮肤只负责未铭刻时的公共名称和说明。

武器和装备统一使用 `item.armament` 标签并进入武器空间或穿戴区。战利品、任务物资和活动物资必须
声明 `storage.backpack` 及单位空间占用后才能进入背包。

首批探险战利品共 `180` 种：十三个区域各六种区域产物、六十种常规敌人凭证、当前三十位探险首领
专属遗物和十二种世界奇珍。它们全部是可堆叠物品，占用一点背包空间，并通过类型化
`ItemRecycleValue` 保存主货币整数回收价。太玄界与魔法世界各自提供完整名称和说明，稳定 ID、掉率、
空间与价格不随世界皮肤变化。

地点 ID 使用 `_x<横坐标>_y<纵坐标>` 后缀，例如 `location.main_city_x0_y0`。正数编码为 `p12`，
负数编码为 `n12`。真实坐标始终读取 `WorldLocationDefinition.x/y`，禁止解析 ID 参与距离计算；正式地点
定义会校验后缀与真实坐标一致。

正式武器首批包含七十二个稳定定义。每把武器拥有独立核心 Property、可执行 Ability、目标选择白名单、
兼容随机属性池、五品质成长表和双世界名称。主机制、辅助机制与目标模式的三元组合不得重复；新机制
必须同时补齐执行原子、静态价值映射、真实触发测试和世界皮肤展示。

武器、装备族与套装遵守统一的软命名规则：约 `70%` 直接采用神话、志怪、民间传说、历史文学和
公共奇幻原型，约 `20%` 使用经典原型与机制称号组合，约 `10%` 留给世界剧情原创。该比例用于指导
整批内容的文化辨识度，不进入规则模型或运行时校验；稳定 ID 永远不因展示名来源而改变。

武器品质不是写死掉率。当前价值区间为黄 `0-62`、玄 `62-74`、地 `74-84`、天 `84-100`、圣
`100+`。固定种子抽检 `9216` 个实例时，当前分布约为 `45.4% / 31.0% / 14.8% / 7.8% / 1.2%`；
后续修改数值或机制后必须重新运行 `test/official_weapon_catalog_test.py`。

正式装备包含十二个纯展示底座族，每族覆盖头部、身体、手部、腰部、足部和饰品六槽，共七十二个
基础装备定义。底座名称不锁流派和随机属性；同名装备实例通过永久资产 ID、品质、词条滚值和套装印记
区分。

装备共享四十八种开放随机词条，每件生成 `2-5` 条。属性词条直接使用人物与战斗属性曲线，机制词条
监听命中、暴击、受伤、闪避、格挡、破盾、击败、回合开始和有效治疗等正式事件。只有火、毒、寒三种
命中附加互斥，其余组合交给完整价值和硬上限约束。

装备词条产生的追加伤害统一携带 `damage.proc` 标签，伤害事实投影为 `is_proc=1`；命中追加、吸血、
受伤恢复等可能形成环路的 Trigger 只响应 `is_proc=0` 的原始伤害。禁止依靠触发次数异常来终止递归。

装备品质区间为黄 `0-23`、玄 `23-32`、地 `32-40`、天 `40-50`、圣 `50+`。固定种子审计
`9216` 个实例时，当前分布约为 `45.8% / 29.4% / 16.0% / 7.4% / 1.4%`，四十八种词条全部
出现且平均生成尝试为 `1.0`。

十二套套装使用 `2/3/4` 件阈值，允许 `4+2`、`3+3` 和 `2+2+2`。套装印记默认以 `25%` 概率
独立生成，不受装备品质影响；装备不存在等级、经验、强化、重铸、宝石、孔位或附魔。

正式敌人内容包含三种阶位、三种遭遇范围、三十二个可组合行为、六十个普通身份和六十个首领身份。
敌人 Ability 使用独立展示 ID，但直接复用正式武器机制已经登记的 Effect、目标规则和触发链。太玄界
使用神话志怪名称，魔法世界使用公共神话与奇幻原型；精英前缀由实例种子稳定选择，首领全部使用固定专名。
组件必须通过 `content.catalog.enemy_projector` 构建战斗实体，通过角色当前
`world_view.enemy_projector` 构建玩家可见名称。

## 固定边界

- 可调数值、品质、物品类型和掉落引用只存在于稳定名录；执行算法存在于规则层。
- 玩家可见名称、说明、图标和别名只存在于世界皮肤。
- 命令组件只使用组合根缓存的角色级 `OfficialContent`，不能自行调用装配器或创建私有目录。
- 两套皮肤必须完整覆盖全部 `display_content_ids`。任意一套缺失都会导致启动装配失败。
- 切换皮肤只替换 `SkinProjector`，不迁移角色、资产或规则状态。
- `PLAYABLE_WORLD_SKIN_IDS` 只声明允许新角色降临和主动跃迁的世界；退出名单的历史皮肤仍可继续登记和还原。

## 使用方式

应用组合根在启动时只装配一次规则目录，再为每套世界皮肤缓存展示视图：

```python
from game.content import (
    PLAYABLE_WORLD_SKIN_IDS,
    WorldViewCatalog,
    assemble_official_catalog,
)

catalog = assemble_official_catalog()
world_views = WorldViewCatalog(catalog, PLAYABLE_WORLD_SKIN_IDS)
content = world_views.require(character_dimension.skin_id)
```

组件通过稳定 ID 读取规则，通过投影器读取展示：

```python
definition = content.catalog.items.require("item.example")
name = content.projector.name(definition.id)
```

武器与装备实例必须通过 `content.gear_projector` 生成完整名称。修仙界使用“品阶品·名称”，魔法世界
使用“稀有度·名称”；随机生成实例把 `intrinsic_value.total` 展示为当前皮肤的评分术语，不得称为售价
或货币价值。铭刻名默认覆盖整个完整名称，而不是继续在铭刻名后附加品质原名。

测试或一次性工具可以使用 `build_official_content()`。正式组件不得重复装配运行期，也不能使用服务器
默认皮肤替代当前角色界相。
