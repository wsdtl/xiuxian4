# 正式内容层

`game/content/` 是具体游戏的次核心内容层。它依赖 `game.core.gameplay` 的公共定义，但不能依赖
命令、消息协议、通信驱动器、账号实现或数据库。

```text
game/content/
├─ catalog/
│  ├─ character/    人物定义与境界规则
│  ├─ combat/       战斗定义、公共数值与价值映射
│  ├─ item/         消耗品等物品定义
│  ├─ weapon/       武器定义、蓝图、机制与平衡审计
│  ├─ equipment/    装备定义、蓝图、随机属性与平衡审计
│  ├─ world/        世界空间与地点定义
│  ├─ foundation.py 跨领域稳定基础定义
│  └─ package.py    汇总各领域名录
├─ world_skins/
│  ├─ cultivation/  基础修仙界，按人物、战斗、物品、武器和世界拆分
│  ├─ magic/        魔法世界，保持同样的领域结构
│  ├─ validation.py 世界皮肤共享的展示完整性校验
│  └─ package.py    汇总全部官方世界皮肤
└─ official.py      统一装配名录并选择世界皮肤
```

`game/core/gameplay/content/skins.py` 只定义 `SkinEntry`、`SkinPack`、`SkinCatalog` 和
`SkinProjector` 等协议中立契约；`game/content/world_skins/` 只提供当前游戏的具体展示数据。两层不能
混放，也不能把具体世界名称写回核心契约。

后续新增材料、消耗品、武器或装备时，先进入 `catalog/` 下对应领域包，再由 `package.py` 汇总；
随后在每个 `world_skins/<世界>/` 的同类模块补齐名称、说明和图标。禁止把全部定义继续堆进
`foundation.py`、单个世界文件或 `package.py`。同一领域一旦同时出现定义、蓝图、机制、属性、价值或
平衡审计等多种职责，必须在领域包内拆分模块，不得重新摊回 `catalog/` 根目录。根 `catalog` 入口只
导出稳定 ID、内容包和通用查询函数，内部编译器与审计器必须从具体领域子模块显式导入。

## 当前内容

```text
content.catalog.base@3.0.0
content.world_skins.official@3.1.0
skin.cultivation@5  基础修仙界
skin.magic@5        魔法世界
```

名录包已经登记主货币、五档品质、公共角色属性、默认人物、一百级成长表、基础攻击、六种药物、固定
新手剑、主世界和原点城市。主货币在基础修仙界显示为“灵石”，在魔法世界显示为“魔晶”；同一个原点
城市分别显示为“太玄仙城”和“星辉王城”。正式组件需要新内容时，先在名录中增加稳定 ID 和规则定义，
再同时为两套世界皮肤补齐名称、说明、图标和命令别名。

品质按规则排序固定为五档。基础修仙界由低到高显示为“黄、玄、地、天、圣”，魔法世界显示为
“普通、精良、稀有、史诗、传说”。品质属于武器和装备实例，不附加到固定消耗品。

基础药物分为小、中、大三档，分别恢复对应最大资源的 `12%`、`25%`、`50%`。修仙界显示为小还丹、
中还丹、大还丹和小回灵丹、中回灵丹、大回灵丹；魔法世界投影为对应的生命药剂和魔力药剂。角色创世
仍只发放两种小药，中药和大药由后续玩法产出。

药物带 `storage.special` 标签并进入纳戒；武器和装备统一使用 `item.armament` 标签并进入武器空间或
穿戴区。战利品、经济物资和跑商货物必须声明 `storage.backpack` 及单位空间占用后才能进入背包。

地点 ID 使用 `_x<横坐标>_y<纵坐标>` 后缀，例如 `location.main_city_x0_y0`。正数编码为 `p12`，
负数编码为 `n12`。真实坐标始终读取 `WorldLocationDefinition.x/y`，禁止解析 ID 参与距离计算；正式地点
定义会校验后缀与真实坐标一致。

正式武器首批包含七十二个稳定定义。每把武器拥有独立核心 Property、可执行 Ability、目标选择白名单、
兼容随机属性池、五品质成长表和双世界名称。主机制、辅助机制与目标模式的三元组合不得重复；新机制
必须同时补齐执行原子、静态价值映射、真实触发测试和世界皮肤展示。

武器品质不是写死掉率。当前价值区间为黄 `0-62`、玄 `62-74`、地 `74-84`、天 `84-100`、圣
`100+`。固定种子抽检 `9216` 个实例时，当前分布约为 `45.4% / 31.0% / 14.8% / 7.8% / 1.2%`；
后续修改数值或机制后必须重新运行 `test/official_weapon_catalog_test.py`。

正式装备包含十二个纯展示底座族，每族覆盖头部、身体、手部、腰部、足部和饰品六槽，共七十二个
基础装备定义。底座名称不锁流派和随机属性；同名装备实例通过永久资产 ID、品质、词条滚值和套装印记
区分。

装备共享四十八种开放随机词条，每件生成 `2-5` 条。属性词条直接使用人物与战斗属性曲线，机制词条
监听命中、暴击、受伤、闪避、格挡、破盾、击败、回合开始和有效治疗等正式事件。只有火、毒、寒三种
命中附加互斥，其余组合交给完整价值和硬上限约束。

装备词条产生的追加伤害统一携带 `damage.proc` 标签，伤害事实投影为 `is_proc=1`；命中追加、吸血、
受伤恢复等可能形成环路的 Trigger 只响应 `is_proc=0` 的原始伤害。禁止依靠触发次数异常来终止递归。

装备品质区间为黄 `0-23`、玄 `23-32`、地 `32-40`、天 `40-50`、圣 `50+`。固定种子审计
`9216` 个实例时，当前分布约为 `45.8% / 29.4% / 16.0% / 7.4% / 1.4%`，四十八种词条全部
出现且平均生成尝试为 `1.0`。

十二套套装使用 `2/3/4` 件阈值，允许 `4+2`、`3+3` 和 `2+2+2`。套装印记默认以 `25%` 概率
独立生成，不受装备品质影响；装备不存在等级、经验、强化、重铸、宝石、孔位或附魔。

## 固定边界

- 规则、数值、品质、物品类型和掉落引用只存在于稳定名录。
- 玩家可见名称、说明、图标和别名只存在于世界皮肤。
- 命令组件只接收组合根注入的 `OfficialContent`，不能自行调用装配器或创建私有目录。
- 两套皮肤必须完整覆盖全部 `display_content_ids`。任意一套缺失都会导致启动装配失败。
- 切换皮肤只替换 `SkinProjector`，不迁移角色、资产或规则状态。

## 使用方式

应用组合根在启动时装配一次：

```python
from game.content import assemble_official_catalog, select_world_skin

catalog = assemble_official_catalog()
content = select_world_skin(catalog, "skin.cultivation")
```

组件通过稳定 ID 读取规则，通过投影器读取展示：

```python
definition = content.catalog.items.require("item.example")
name = content.projector.name(definition.id)
```

测试或一次性工具可以使用 `build_official_content()`。正式组件不得重复装配运行期。
