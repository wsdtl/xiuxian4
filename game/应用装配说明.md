# 游戏应用装配

本模块集中组装正式内容、账号服务、角色创世服务和 SQLite 数据库。命令组件只读取已经组装好的服务，
不能自行创建数据库连接、内容运行期或仓储。

通信驱动器提供协议中立的 `MessageIdentity`；本层把它转换为账号底座的 `IdentityEvidence`。因此命令
组件不需要按 QQ、本地或未来驱动器分别注册。

本文件属于具体游戏，不属于 `auto/`。`auto/` 只承载框架自动初始化；启动配置通过
`ROUTER_MODULES=["game.app"]` 显式加载本模块。

启动生命周期会在接收消息前调用 `database.initialize()`。测试使用临时数据库时，可以通过
`build_game_services()` 构造独立服务，并用 `install_game_services()` 和 `restore_game_services()`
控制当前服务集合。

账号外部身份在写入数据库前会使用 `ACCOUNT_IDENTITY_SECRET` 做 HMAC 保护；生产配置必须使用独立、
不可提交到 Git 的随机密钥，不能复用机器人密钥。
