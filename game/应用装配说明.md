# 游戏应用装配

本模块集中组装正式内容、账号服务、角色创世服务、物品使用服务、铭刻服务、装配服务、行动服务、
休息、探险、跨界灾厄与抽奖玩法、通知收件箱、全服活动服务和 SQLite 数据库。命令组件只读取已经组装好的服务，不能
自行创建数据库连接、内容运行期或仓储。

`GameServices` 是命令层稳定门面，不是业务实现层。允许保留异常映射和一行转发方法，但不得直接开启
数据库工作单元或构造领域事务；角色入口与个人读模型归 `PlayerFeature`，跃迁联合事务归
`DimensionShiftFeature`，其他玩法分别归自己的 `game/features/<domain>`。

定时任务触发器归各自的中文二级组件 `jobs.py`，由组件导入时注册；触发器只负责调度和调用 feature，
不在组合根集中维护任务，也不在触发器中实现玩法规则。

通信驱动器提供协议中立的 `MessageIdentity`；本层把它转换为账号底座的 `IdentityEvidence`。因此命令
组件不需要按 QQ、本地或未来驱动器分别注册。

本文件属于具体游戏，不属于 `auto/`。`auto/` 只承载框架自动初始化；启动配置通过
`ROUTER_MODULES=["game.app"]` 显式加载本模块。

启动生命周期会在接收消息前调用 `database.initialize()`。测试使用临时数据库时，可以通过
`build_game_services()` 构造独立服务，并用 `install_game_services()` 和 `restore_game_services()`
控制当前服务集合。

`load_player_reply_state()` 按当前消息身份只读汇总角色、个人设置、活动热点、未读通知数量和待领取行动
数量，供命令层统一生成人物头、活动通栏与提醒通栏。`load_global_activity_views()` 列出全部开放且已注册
的全服活动。这些读取不会标记通知、领取行动、参与活动或修改任何业务快照。

铭刻与装配分别使用 `PersistedInscriptionService` 和 `PersistedLoadoutService`。铭刻之羽消费、实例改名、
装配槽位和库存位置都在各自唯一的 SQLite 工作单元内提交。角色设置和通知已读由 `GameServices` 再做
当前角色、当前账号边界校验，命令组件不能直接调用快照仓储。

手动物品使用通过 `PersistedItemUseService` 原子扣除库存并写回人物资源。使用前会组合当前武器、装备和
套装贡献，保证恢复药按照真实资源上限结算。铭刻之羽拥有独立保管容器，不进入纳戒或普通消耗品流程。
武器成长道具通过 `PersistedWeaponItemUseService` 联合提交道具扣除、武器聚合与武库实例状态，失败时
三者全部回滚。
芥子神砂通过 `SpecialItemUseService` 提交，背包扩容与库存扣除处于同一事务。
跃迁凭证由 `DimensionShiftFeature` 在跃迁工作单元中自动扣除；凭证扣除、角色 `world_id` 更新和存在体
跨 `space_id` 迁移必须同时成功。查看界门、失败、原地和主要行动占用不得消耗或改变位置。

启动生命周期会为 `activity.scope.global` 初始化唯一活动快照。活动注册目录在应用服务组装时复制并
校验，保证具体活动组件不能在服务运行中悄悄改变热点规则。

`DimensionalDisasterFeature` 复用内容周期、活动状态机、角色与敌人投影、公共战斗时间线和统一奖励
结算。共享血量、每日次数、挑战回执和遗羽结算进度保存在独立业务聚合；贡献排行保存在全服活动实例。
调度器每分钟开放当前窗口或处理到期封榜，奖励重试依靠统一领取凭据防止重复经验和第二枚遗羽。

`ExplorationFeature` 由本组合根注入数据库工作单元、共享快照 codec、统一奖励结算和库存引擎。
它本身不导入 SQLite 实现。调度器每分钟发现一次已经到期的持续探险，并以十分钟为一个稳定
批次补算；玩家查询和停止时也会走同一结算入口，因此服务重启不会制造第二套补偿逻辑。

`EconomyFeature` 复用同一个库存引擎、账本引擎和工作单元。归航回收先生成不可变报价，再在同一
事务中注销物品档案并由主货币发行账户向角色钱包入账，不扣除归航库余额，也不收交易税。归航市场
统一处理武器装备实例和固定数量堆叠物品，分别提交卖方预约库存、买方接收库存、卖方货款与归航公约
税金；数据库异常不会留下只转物、只扣款或漏税的半成品。

`LotteryFeature` 使用独立彩票快照，并复用同一个账本引擎。每个角色每期只能购买一个号码，号码和
票款在同一事务提交；后台每分钟补开到期轮次，按六位环形号码距离排名，并在同一工作单元内提交开奖记录、
归航库扣款和中奖钱包入账。至少两人参与才开奖，人数不足自动退票；资金异常不足时轮次保持待开奖，
不会作废彩票或额外发行货币。

`DrawFeature` 读取正式内容包冻结的奖池，在同一个工作单元内扣除抽奖签、推进共享保底、发放恢复药
或主货币并写入最近一百次幂等历史。重复消息直接返回原回执；库存、账本或奖励结算任一步失败都会回滚
扣签。抽奖演出只属于命令展示，GIF 缺失不会影响已经完成的结算。

`RestFeature` 使用公共主行动槽，并在一个工作单元内提交行动状态、人物恢复和累计恢复窗口。后台每分钟
完成已经达到三十分钟的休息；主动停止按累计实际休息时间结算，重复启停不会重复获得首分钟恢复。
休息开始时检查持续探险，探险开始时检查主行动槽，双方保持互斥。

账号外部身份在写入数据库前会使用 `ACCOUNT_IDENTITY_SECRET` 做 HMAC 保护；生产配置必须使用独立、
不可提交到 Git 的随机密钥，不能复用机器人密钥。
