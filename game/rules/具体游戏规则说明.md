# 具体游戏规则

`game/rules/` 只保存多个组件可能复用的游戏内部规则策略，目录与模块统一使用英文。玩家命令、
组件业务和回复展示放在同一个 `game/cmd/<中文组件>/` 目录中：`__init__.py` 只注册命令，
`service.py` 调用本层规则及其他底座。

## 当前角色策略

- 一个账号只能拥有一个角色。
- 创建时显式指定的名称优先。
- 未指定名称时使用消息驱动提供的平台名称。
- 平台事件没有名称时必须要求玩家显式指定，禁止把 OpenID 当作角色名。
- 名称只允许中文、英文字母和数字，最大显示宽度为 12；暂不要求全服唯一。

```python
identity = CharacterIdentityPolicy().prepare_creation(
    account_id=account_id,
    requested_name=command_argument,
    platform_name=message_context.sender_name,
    existing_character_ids=repository.character_ids_for(account_id),
)
```

角色创世服务必须在同一个写事务内读取账号已有角色、执行本策略并保存全部初态，才能保证并发创建
不会突破一个账号一个角色。

## 人物数值策略

`game.rules.character.initialization` 固定本游戏的裸装起点：

```text
最大血气 100
最大灵力 100
基础攻击 10
基础防御 0
行动速度 100
```

标准人物等级每提升一级只增加最大血气 `10` 和基础攻击 `1`。灵力、防御和速度不随等级自动增长，
由武器、装备、套装和其他明确来源塑造。提高最大血气时当前血气同步增加相同值，保持缺失血气不变。

## 角色创世策略

- 初始 `Lv1`、经验 `0`，人物等级上限 `100`。
- 角色自带基础攻击，因此卸下全部武器装备后仍可以进入战斗。
- 初始背包容量 `40`，基础货币 `100`，小型血气药和小型灵力药各 `2`。
- 初始获得一把固定凡品 `Lv1` 新手剑，不获得装备。
- 配装编号固定为 `0-5` 共六套；默认激活 `0`，新手剑只属于配装 `0`，其余五套为空。
- 自动用药默认开启。血气在 `25%` 以下尝试恢复至 `55%`，灵力在 `15%` 以下尝试恢复至 `45%`。
- 休息至少一分钟并恢复缺失量的 `50%`，三十分钟恢复全部缺失量；同一恢复窗口不能重复领取首分钟收益。

`CharacterCreationPlanner` 只生成完整初态，不接触数据库。`PersistedCharacterCreationService` 在同一个
SQLite 写事务中提交角色目录、人物、背包、配装、设置、公共账本和公共世界。重放同一事务返回原回执，
任意中途故障整单回滚。

人物升级经验以 `99` 个明确整数保存在正式内容层。`Lv1-Lv80` 使用成熟曲线，后十九级把单级需求
增幅从 `6%` 平滑压到 `2%`，`Lv100` 累计需要 `270305413` 经验。

境界由人物等级实时推导，不保存第二份角色状态：`Lv1-Lv90` 每十级一个境界，`Lv91-Lv100` 每一级
一个独立境界，共十九境。世界皮肤提供完整名和最大显示宽度为 `8` 的短名；人物头统一显示
`境界短名 角色名 Lv等级`，角色详情显示完整境界名。

## 全服活动策略

`game.rules.activity` 是具体活动组件与统一活动入口之间的注册边界。具体业务调用
`register_global_activity()` 声明活动定义、热点优先级、开放初期和结束前的展示窗口；回复层只读取注册
结果，不识别未声明的活动实例。

- 全服活动统一使用作用域 `activity.scope.global`，数据库只保存一份活动状态。
- “开放”与“顶部热点”是两个状态：完整活动页展示全部开放活动，顶部只展示当前热点。
- 默认在开放后 `12` 小时和结束前 `12` 小时进入顶部；短活动两段窗口重叠时可以全程展示。
- 顶部按优先级、结束时间、开放时间和实例 ID 稳定排序，最多显示两个活动，其余显示为 `+N`。
- 每个注册活动必须在当前世界皮肤提供 `compact_name`，最大显示宽度为 `8`。
- 详情入口只读取稳定活动实例 ID；查看列表或详情不会参与、领奖或推进生命周期。
