# 具体游戏规则

`game/rules/` 只保存多个组件可能复用的游戏内部规则算法，目录与模块统一使用英文。可调概率、阈值、
初始资产和成长数值由 `game/content/catalog` 提供，本层只能读取，不能重新声明。玩家命令、
组件业务和回复展示放在同一个 `game/cmd/<中文组件>/` 目录中：`__init__.py` 只注册命令，
`service.py` 调用本层规则及其他底座。

## 当前角色策略

- 一个账号只能拥有一个角色。
- 创建时显式指定的名称优先。
- 未指定名称时使用消息驱动提供的平台名称。
- 平台事件没有名称时必须要求玩家显式指定，禁止把 OpenID 当作角色名。
- 名称只允许中文、英文字母和数字，最大显示宽度为 12；暂不要求全服唯一。

```python
identity = CharacterIdentityPolicy().prepare_creation(
    account_id=account_id,
    requested_name=command_argument,
    platform_name=message_context.sender_name,
    existing_character_ids=repository.character_ids_for(account_id),
)
```

角色创世服务必须在同一个写事务内读取账号已有角色、执行本策略并保存全部初态，才能保证并发创建
不会突破一个账号一个角色。

## 人物数值策略

`game.content.catalog.character.definitions` 固定本游戏的裸装起点：

```text
最大血气 100
最大灵力 100
基础攻击 10
基础防御 0
行动速度 100
```

标准人物等级每提升一级只增加最大血气 `10` 和基础攻击 `1`。灵力、防御和速度不随等级自动增长，
由武器、装备、套装和其他明确来源塑造。提高最大血气时当前血气同步增加相同值，保持缺失血气不变。

## 玩家战斗投影策略

`game.rules.combat.PlayerCombatProjector` 是角色当前配装进入战斗的唯一构筑入口。它校验角色、库存与配装
归属，按当前武器和六个装备槽汇总武器品质及等级、随机词条、装备品质、套装门槛和角色永久特征，再交给
公共 `CharacterProjector` 生成最终规则实体。

探险、多次元灾厄和战斗面板必须持有应用组合根装配的同一个 `PlayerCombatProjector`。场景只能额外提供
场景标签或明确贡献，不能复制武器、装备、套装与属性求值代码；命令展示只能读取最终属性快照，不能另算
一套玩家面板。

## 角色创世策略

- 初始 `Lv1`、经验 `0`，人物等级上限 `100`。
- 角色自带基础攻击，因此卸下全部武器装备后仍可以进入战斗。
- 初始背包空间 `40`，基础货币 `100`，小型血气药和小型灵力药各 `2`。
- 纳戒保存药物和特殊物品，铭刻系统独立保管铭刻之羽，武器空间保存武器与六类装备，背包保存战利品、任务物资和活动物资；
  只有背包按物品单位空间限制容量。
- 初始获得一把固定黄档 `Lv1` 新手剑，不获得装备。
- 配装编号固定为 `0-5` 共六套；默认激活 `0`，新手剑只属于配装 `0`，其余五套为空。
- 自动用药默认开启。血气在 `25%` 以下尝试恢复至 `55%`，灵力在 `15%` 以下尝试恢复至 `45%`。
- 休息至少一分钟并恢复缺失量的 `50%`，三十分钟恢复全部缺失量；同一恢复窗口不能重复领取首分钟收益。
- 从全部启用世界皮肤中做一次可重放的随机降临，并把界相状态与角色其他初态放在同一个创世事务中。

`CharacterCreationPlanner` 只生成完整初态，不接触数据库。`PersistedCharacterCreationService` 在同一个
SQLite 写事务中提交角色目录、人物、三类物品空间、穿戴区、配装、设置、公共账本和公共世界。重放同一事务返回原回执，
任意中途故障整单回滚。

## 角色界相与跃迁策略

- `CharacterDimensionState` 独立保存角色当前 `skin_id`、抵达时间和 revision，不进入公共 `CharacterState`。
- 一个世界皮肤就是一套玩家可见界相；底层角色、资产、地点、敌人和规则仍使用唯一稳定 ID。
- `WorldViewCatalog` 复用启动期已经冻结的规则目录，按 `skin_id + version` 缓存人物、物品、装备和敌人投影。
- 创建角色时通过创世 `RuleContext.random` 从 `PLAYABLE_WORLD_SKIN_IDS` 随机选择界相；同一事务重放必须返回相同结果。
- `跃迁 世界名称` 只更新界相快照。相同目标幂等，空闲时免费执行，主要行动进行中拒绝执行。
- 组件不得读取服务器默认皮肤生成玩家正文，必须根据当前角色界相取得世界视图。
- 探险和其他可长期保存的业务状态只保存稳定 ID 或结构化实例引用，不能提前冻结玩家可见名称。
- 铭刻名高于世界投影；开启原名显示时，括号内的完整原名使用当前角色界相重新生成。

人物升级经验以 `99` 个明确整数保存在正式内容层。`Lv1-Lv80` 使用成熟曲线，后十九级把单级需求
增幅从 `6%` 平滑压到 `2%`，`Lv100` 累计需要 `270305413` 经验。

境界由人物等级实时推导，不保存第二份角色状态：`Lv1-Lv90` 每十级一个境界，`Lv91-Lv100` 每一级
一个独立境界，共十九境。世界皮肤提供完整名和最大显示宽度为 `8` 的短名；人物头统一显示
`境界短名 角色名 Lv等级`，角色详情显示完整境界名。

## 装备实例策略

`EquipmentInstanceGenerator` 使用正式装备定义绑定的开放生成策略创建 `2-5` 条随机词条，品质在全部
词条完成估值后确定。随后独立进行一次套装印记判定：默认概率为 `25%`，命中后从十二套套装中等概率
选择一套。`25%` 由内容层提供，规则生成器只负责执行判定。套装概率不读取品质，因此黄品套装装备也
可能具有保留价值。

装备生成使用同一个 `RuleContext` 和确定性随机源，任何异常都会恢复随机检查点。结果保存完整
`ItemRollState`、生成凭据、套装判定值和最终 `EquipmentState`；掉落、活动和兑换组件只负责选择具体
装备定义及资产 ID，不能自行滚词条或决定品质。

## 武器实例策略

`WeaponInstanceGenerator` 与装备生成器共用同一个 `ItemizationEngine`。它只接受具有武器生成策略的
正式武器定义，一次产生唯一核心特色、兼容随机属性、最终品质和可持久化的 `WeaponState`。固定新手
武器没有生成策略，不能通过随机生成器产出。

## 全服活动策略

`game.rules.activity` 是具体活动组件与统一活动入口之间的注册边界。具体业务调用
`register_global_activity()` 声明活动定义、热点优先级、开放初期和结束前的展示窗口；回复层只读取注册
结果，不识别未声明的活动实例。

- 全服活动统一使用作用域 `activity.scope.global`，数据库只保存一份活动状态。
- “开放”与“顶部热点”是两个状态：完整活动页展示全部开放活动，顶部只展示当前热点。
- 默认在开放后 `12` 小时和结束前 `12` 小时进入顶部；短活动两段窗口重叠时可以全程展示。
- 顶部按优先级、结束时间、开放时间和实例 ID 稳定排序，最多显示两个活动，其余显示为 `+N`。
- 跟随世界表象的活动必须在全部可进入世界皮肤提供 `compact_name`；多次元灾厄这类源层活动可以在注册时
  提供固定展示。两者短名最大显示宽度均为 `8`。
- 详情入口只读取稳定活动实例 ID；查看列表或详情不会参与、领奖或推进生命周期。

默认窗口与顶部数量由 `game.content.catalog.activity` 提供。本层的 `ActivitySpotlightPolicy` 和
`GlobalActivityCatalog` 只负责时间判断、排序和截取，不拥有这些可调数字。

## 多次元灾厄策略

- 多次元灾厄不属于任何世界皮肤；所有登录者看到同一个正式名称、故事、能力身份和遗羽文案。
- 每套启用世界皮肤为灾厄池贡献素材，抽取时先选来源再选灾厄，避免内容数量较多的来源挤压其他来源。
- 每周开放两个互不重叠的 `48` 小时窗口；同一时间全服只存在一个活动实例、共享血量和贡献排行。
- 玩家每天最多讨伐两次，不设实时冷却。必须停止探险且血气大于零，战后真实血气和灵力写回角色。
- 单次战斗使用当前武器、六槽装备、随机词条、套装和全部公共战斗特效；灾厄只编排场景，不复制公式。
- 贡献等于累计有效伤害。到期后自动封榜，有效参与者按击破结果、贡献平方根和小幅排名加成获得经验。
- 未击破的灾厄退去，不产生铭刻之羽；击破事件按最低贡献、平方根权重和 `35%` 权重上限抽取一名持有者。
- 每个击破事件全服只生成一枚遗羽。遗羽保存唯一资产 ID、事件 ID、降临日期、参与人数、最初持有者
  贡献和排名；灾厄复临复用固定故事，但不能复用事件遗羽。
- 次元跃迁不修改灾厄血量、次数、贡献、排行或遗羽身份。

## 敌人与遭遇策略

- 玩家可见类型为普通、精英、首领和多次元灾厄；底层使用普通/精英/首领阶位与个人/队伍/全服范围组合。
- 区域决定遭遇允许的等级范围，不把所有低级区域强制同步到玩家等级。
- 普通遭遇生成一至三名敌人，精英遭遇生成一名携带两至三个兼容行为的精英。
- 敌人不拥有玩家库存和装备实例；行为模板通过公共 Contribution 与共享 Ability/Effect 获得战斗能力。
- 普通与精英实例只保存于本次遭遇凭据，多次元灾厄的共享血量使用独立业务聚合，参与贡献交给全服活动状态。
- `EnemyDefeatRewardPlanner` 只报价人物经验、武器经验和掉落表请求，不能直接修改角色、库存或账本。

## 战利品与收购策略

- 胜利后区域战利品必得；药物、武器装备和世界奇珍使用彼此独立的掉落组。
- 普通敌人掉一份区域产物，精英掉两份，首领掉五份并附带一件专属遗物。
- `ItemSaleValue` 的整数单价属于内容层；`game.rules.item` 只读取库存与名录生成不可变报价。
- 一次出售只处理背包内带 `item.trophy` 与 `loot.sellable` 标签的未占用堆叠，不触碰药物和装备。
- 玩法层在同一工作单元中扣物并向角色钱包入账，任意一步失败都会整体回滚。
