# 装备底座说明

## 封板版本

```text
equipment.foundation.v1
```

装备负责辅助、转化和连接武器战斗循环。玩家同时可以激活六件装备。

## 结构

```text
槽位 + 流派 + 生成后品质 + 开放随机属性 + 可选套装身份
```

装备定义固定槽位、流派和可选套装身份。生成型装备实例保存品质与完整 `ItemRollState`，品质由
随机属性完成后的内在价值区间决定，不是生成前指定。

装备明确没有：

```text
等级
经验
强化
宝石
孔位
附魔
技能书
```

装备随机属性不执行武器流派兼容约束。只要属性和机制已经登记价值，就允许跨方向组合；品质区间
负责控制单件总价值，运气决定具体组合。

## 流派与品质

流派决定机制方向，同一流派在六个槽位可以提供不同但互相连接的贡献。每件装备自身贡献可以独立
工作，同流派和混搭都能直接工作。`EquipmentSetDefinition` 可以额外声明按件数递增的套装贡献，
套装价值不计入单件品质。

品质同时是内在价值区间的结果和内容目录中的稳定身份。品质档案可以提供少量固定贡献，随机贡献
仍以生成凭据为准：

```text
普通：触发率 0.15
优秀：触发率 0.22
稀有：触发率 0.30
珍贵：触发率 0.40
```

同一品质允许出现不同组合；低品质也可能因机制协同在特定整套配装中产生高边际价值。品质区间和
固定贡献都由内容包显式声明，装备引擎不使用隐藏公式。

## 角色贡献

`EquipmentContributionProvider` 合并流派标签、装备基础贡献、品质贡献和经重新校验的随机贡献。
`set_contributions()` 按当前激活装备另外产生套装贡献。属性、Ability、Trigger 和伤害干预仍由
Gameplay 内核处理。

完全相同属性的装备仍可作为独立实例保存来源。聊天输出应先展示品质和关键属性，再由次核心决定
是否折叠真正相同的实例；不能只按“槽位 + 流派 + 品质”吞掉随机差异。

## 不属于本底座

- 具体流派、装备和品质数值。
- 装备等级与经验。
- 附魔、宝石、孔位和强化。
- 掉落、交易、自动回收和展示文案。
- 实例铭刻由独立铭刻底座承接。
