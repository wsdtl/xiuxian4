# 装备底座说明

## 封板版本

```text
equipment.foundation.v2
```

装备负责辅助、转化和连接武器战斗循环。玩家同时可以激活六件装备。

## 结构

```text
槽位 + 底座族 + 生成后品质 + 开放随机属性 + 实例套装印记
```

装备定义只固定槽位和有限底座族。生成型装备实例保存品质、完整 `ItemRollState` 和可选套装印记，
因此同一个装备名字可以随机出不同属性和不同套装身份。品质由随机属性完成后的内在价值区间决定，
不是生成前指定。

装备明确没有：

```text
等级
经验
强化
宝石
孔位
附魔
技能书
```

装备随机属性不执行武器流派兼容约束。只要属性和机制已经登记价值，就允许跨方向组合；品质区间
负责控制单件总价值，运气决定具体组合。

## 底座族与品质

底座族只提供有限装备外形和展示身份，不提供固定流派倾向，不锁随机属性池，也不绑定套装。随机属性原则上跨方向
开放，只排除重复、绝对冲突、无法触发和突破硬上限的组合。`EquipmentSetDefinition` 声明按件数
递增的套装贡献，具体 `set_id` 由 `EquipmentState` 保存并参与当前装配统计。

品质同时是内在价值区间的结果和内容目录中的稳定身份。品质档案可以提供少量固定贡献，随机贡献
仍以生成凭据为准：

```text
黄：总价值 0-23
玄：总价值 23-32
地：总价值 32-40
天：总价值 40-50
圣：总价值 50+
```

同一品质允许出现不同组合；低品质也可能因机制协同在特定整套配装中产生高边际价值。品质区间和
固定贡献都由内容包显式声明，装备引擎不使用隐藏公式。

正式内容使用十二套 `2/3/4` 件套装。套装印记与品质独立，默认生成概率属于具体游戏内容，不属于
本底座；规则层读取内容概率执行判定，核心只验证套装存在、槽位不重复和当前装配件数。

## 角色贡献

`EquipmentContributionProvider` 合并底座族标签、装备基础贡献、品质贡献和经重新校验的随机贡献。
`set_contributions()` 按当前激活装备另外产生套装贡献。属性、Ability、Trigger 和伤害干预仍由
Gameplay 内核处理。

完全相同属性的装备仍可作为独立实例保存来源。聊天输出应先展示品质和关键属性，再由次核心决定
是否折叠真正相同的实例；不能只按“槽位 + 底座族 + 品质”吞掉随机差异。

## 不属于本底座

- 具体底座族、装备、套装和品质数值。
- 装备等级与经验。
- 附魔、宝石、孔位和强化。
- 掉落、交易、自动回收和展示文案。
- 实例铭刻由独立铭刻底座承接。
