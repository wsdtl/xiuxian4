# 掉落与保底底座说明

## 封板版本

```text
loot.foundation.v1
```

## 边界

掉落底座只决定稳定奖励项和数量，不发放资产，也不知道物品、武器、货币或玩家可见名称。游戏
次核心把 `award_id` 翻译为具体 `RewardSpec`，再交给统一奖励结算。

所有随机数来自 `RuleContext.random`。掉落表和凭据保存版本，测试和审计可以使用相同种子重放。

## 抽取组

- `weighted_one`：按整数权重选择一个条目，显式空条目的 `award_id` 为 `None`。
- `independent`：每个条目按百万分比独立判定，整组可以落空。
- `all`：发放全部符合标签条件的条目。

每组可以重复抽取；加权组可以要求同批不重复。条目支持数量区间、必需标签和阻止标签。

## 修正与保底

命令使用整数基点修正指定条目的权重或概率，不接受浮点百分比。`10000` 表示不变，`0` 表示本次
禁用，修正后的独立概率最高限制为百万分之一百万。

保底只作用于一个加权组，明确区分：

- 哪些条目会重置计数。
- 触发保底时允许从哪些条目中选择。
- 连续多少次没有命中后，下一次强制进入保底候选。

保底计数属于账号或其他领取主体的 `LootState`，不属于掉落表定义。

## 审计凭据

`LootRollReceipt` 保存表 ID、表版本、逻辑时间、每次判定的采样值和尺度、是否强制保底、最终奖励
项、数量和保底前后值。展示层不能从文案反推抽取结果。

## 不属于本底座

- 奖励项到具体资产 ID、容器和经济账户的解析。
- 奖励发放、背包容量、重复领取和跨领域事务。
- 具体概率、奖池内容、活动资格和付费成本。
- 玩家可见名称、动画、命令、按钮和公告。
