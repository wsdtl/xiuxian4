# 事实投影、通知与排名底座说明

## 封板版本

```text
projection.foundation.v1
```

## 永久事实与 Outbox

每次持久化事务追加 Outbox 时，同一数据库事务还会写入 `fact_journal`。Outbox 是可以确认发布的
传输队列；事实日志是追加式长期来源。发布 Outbox 不会删除或修改事实。

事实保存全局递增偏移、事务 ID、事务内序号、稳定事件类型、结构化负载和发生时间。事实不是聚合
状态的替代品，不能通过重放事实偷偷改变已经发布的领域语义。

## 投影

每个投影使用 `projector_id + partition_id` 保存检查点。游戏次核心读取检查点之后的事实，计算明确
更新，再通过 `ProjectionStore.commit()` 把记录变化和检查点推进放进同一个事务。检查点使用 CAS
revision，失败时不会留下半批投影。

投影代码或结构升级时，必须使用 `ProjectionStore.reset()` 携带当前 revision 显式归零分区，再从
`fact_journal` 的 offset 0 重放。归零、清除旧记录和检查点推进属于同一个事务；并发 revision
不一致时全部回滚。读取接口不会隐式重建投影。

读取投影、榜单和通知永远是只读操作，不运行 worker、不清理状态，也不推进活动。

## 通知

通知保存稳定类型、接收主体、事实来源、防重键、优先级、结构化数据和可选动作语义，不保存
Markdown、按钮或具体命令。消息层根据当前驱动能力翻译动作。

通知读取不会自动标记已读。已读和忽略都是带 revision 的显式写操作。过期通知只在查询中过滤，
保留策略由独立维护任务负责。

未读数量统计同样是只读查询，可以用于回复通栏摘要；展示过通知或数量不等于已读。只有显式处理命令
才能调用 `mark()` 改变通知状态。

## 排名快照

排名引擎使用分数、整数并列值和主体 ID 得到完全稳定的顺序。冻结快照包含榜单、作用域、周期、
版本、事实偏移和冻结时间。相同身份只能幂等保存相同内容，不能覆盖历史结果。
排名快照的事实偏移不能越过数据库已经存在的最大事实水位。

## 不属于本底座

- 哪些事实形成日记、称号、早报、统计或通知。
- 榜单候选来源、奖励范围和运营展示。
- Markdown 文案、按钮、主动推送和网页。
- 具体事实和投影的业务保留天数。
