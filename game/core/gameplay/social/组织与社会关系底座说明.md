# 组织与社会关系底座说明

## 封板版本

```text
social.foundation.v1
```

社会底座包含三组相互独立但共享主体身份的能力：

- 组织、成员、角色、权限、容量、排他归属和分类贡献。
- 发送方与接收方之间可接受、拒绝、取消或过期的请求。
- 有向或无向、带整数边界和溢出策略的关系值。

组织类型和角色由内容包声明。领导角色必须拥有解散权限；角色人数上限、组织容量和同类型排他
归属由核心统一校验。加入审批、创建成本、贡献来源和组织成长仍由游戏次核心负责。

领导权使用 `TransferOrganizationLeadership` 在一次状态事务中完成卸任与接任，避免领导角色上限
为 1 时发生状态机死锁。创建者不是永久成员，完成领导权转移后可以正常离开组织。

社会请求只保存结构化身份、期限和状态，不直接执行决斗、交易或邀请结果。接收请求后，游戏次
核心再提交相应战斗、交换或组织命令。

关系值可以承接仇恨、好感、信誉等稳定数值，但具体增减公式、衰减周期和展示名称不属于核心。

## 不属于本底座

- 宗门等级、山门、供养、宗门大会和创建材料。
- 切磋、决斗、婚契、队伍等具体请求结果。
- 仇恨、好感和信誉的业务计算公式。
- 成员列表、邀请按钮、通知和管理页面。
