# 战斗底座封板说明

## 封板版本

```text
combat.foundation.v5
```

代码可通过 `gameplay.combat.COMBAT_FOUNDATION_VERSION` 读取该版本。

封板表示战斗规则职责、执行顺序、公开数据结构和事件语义进入稳定状态。它不表示
具体角色、装备、技能和数值已经完成。

`combat.foundation.v5` 增加核心原生 `BattleSession` 与 `BattleTrace`。所有连续战斗模式必须通过
会话推进状态，不能自行维护初始、回合、行动和最终快照。轨迹逐次保存行动前后完整 `BattleState`、
实际目标和全部 `RuleEvent`，同时保留规则内核与展示层之间的单向边界。

`combat.foundation.v4` 在所有伤害事实中公开 `is_proc`，由发起伤害的操作通过 `damage.proc` 标签
声明追加伤害。命中、受伤和反应类 Trigger 可以只响应原始伤害，避免追加伤害再次触发自身；日志层
只翻译该事实，不参与递归判断。

`combat.foundation.v3` 补齐确定性随机操作分支、Effect 层数公式、当前冷却干预和纯标记状态，允许
武器通过规则组合表达随机高低谷、叠印引爆、禁法与持续状态，不需要在伤害引擎中写武器特例。

`combat.foundation.v2` 将速度从“每轮只决定先后手”升级为行动进度。标准速度为 `100`，速度通过
递减收益曲线决定长期行动频率；一个时间轮内允许同一实体获得多次普通行动，慢速实体也可能隔轮
才能积满一次行动。开场仍保证每名存活参与者拥有一次行动。

## 已冻结边界

以下协议后续默认保持稳定：

- `Ability → Effect → Damage/Recovery/Control → Trigger/Interceptor` 执行关系。
- 属性修改层级和复合数值公式求值方式。
- 单目标、多目标和多段 Effect 的原子执行边界。
- 伤害、治疗、护盾、控制和资源变化的实际值语义。
- 时间轮开始、行动、回合结束、状态到期和胜负顺序。
- `BattleState.action_progress` 的可重放行动进度和确定性排序。
- `RuleContext` 的规则版本、逻辑时间、确定性随机与失败回滚。
- `RuleEvent` 的来源、目标、阶段和追踪元数据。
- 服务端 Ability 目标白名单与目标约束。
- ActiveEffect 的来源、参数、层数和持续时间。
- 退出实体作为历史来源保留，但不能行动、被选择或触发被动。

## 允许直接扩展

以下工作不需要重新打开战斗内核：

- 新增属性、资源和伤害类型定义。
- 新增 Ability、Effect、Trigger 和状态组合。
- 新增武器、装备、体质、技能书和流派内容。
- 新增世界皮肤名称、描述和图标。
- 注册新的数值公式求值器。
- 注册新的 Effect 原子操作。
- 注册新的伤害干预处理器。
- 注册新的目标选择器和目标约束。
- 新增战斗 AI、奖励、掉落、QQ 输出和网页战报。

普通内容不得通过修改 `DamageEngine` 或 `BattleEngine` 添加特例。

## 必须重新审核

只有出现下列情况才重新打开内核：

- 需要增加全新的结算阶段，而不是现有阶段中的新处理器。
- 现有事件无法准确表达新的规则事实。
- 原子执行和失败回滚边界需要变化。
- 一次行动不再能用现有多实体状态表示。
- 确认存在无法通过注册表、定义和组合解决的底层机制。

重新打开时必须：

1. 说明现有扩展点为什么无法承载。
2. 更新封板版本。
3. 更新结构化事件协议和中文说明。
4. 增加成功、失败、重放和边界回归测试。
5. 运行全项目测试与 Gameplay 依赖扫描。

## 不属于封板内容

以下内容尚未建设，但不会要求修改战斗主流程：

- 玩家角色、成长与境界。
- 具体武器、装备、技能和流派数值内容。
- 具体敌人行为组合和首领阶段内容；通用自动决策已经由独立 `combat.ai` 扩展提供。
- 战斗奖励、经验、掉落与经济结算。
- 数据库存储与恢复。
- QQ 战斗指令、摘要和网页战报。

## 验证基线

封板时验证：

- 全部 13 组测试脚本通过。
- `python -m compileall -q gameplay test` 通过。
- `game/core/gameplay/` 不依赖 `launch`、`message`、`components` 或数据库。
- 高级特效测试覆盖复合公式、状态操作、治疗、伤害干预、分摊、控制、行动指令、目标约束和动态参战。
- 战斗时间线测试覆盖群攻计费、周期来源、控制跳过、速度频率、胜负、平局和随机回滚。
