# 战斗编排说明

## 职责

`BattleEngine` 只决定谁在什么时候对哪些目标使用 Ability，并维护一场战斗的
运行状态。它不复制技能、Effect 或伤害公式。

```text
战斗开始
→ 创建本轮速度顺序
→ 行动者回合开始
→ 结算中毒等回合开始触发
→ 检查败退或控制状态
→ 选择目标
→ 执行单目标或多目标 Ability
→ 逐段处理 Trigger
→ 回合结束触发
→ 缩短行动者的 Effect 和冷却
→ 判断胜负
→ 下一行动者或下一轮
```

每一步产生 `RuleEvent`。QQ、网页战报和统计只能读取这些事实，不能反向修改
结算。

## BattleState

`BattleState` 是不可变状态，保存：

- 战斗 id。
- 参与者阵营和站位。
- 全部 `RuleEntity` 当前状态。
- 当前轮数和总行动数。
- 本轮行动顺序和当前行动者。
- 已退出但为历史来源保留的实体集合。
- 进行中、结束或平局状态。
- 获胜阵营和状态修订号。

执行失败不会返回半成品状态，随机游标也会回滚。数据库以后只负责持久化该
状态，不参与规则计算。

## 行动顺序

配置速度属性后，每轮开始按速度从高到低生成顺序；同速时按阵营、站位和实体
id 稳定排序。未配置速度属性时使用稳定站位顺序。

败退实体不会进入下一轮。达到 `maximum_rounds` 仍有多个阵营存活时判为平局，
避免异常构筑制造永久战斗。

## 目标选择

目标算法使用可注册的稳定 id。当前内置：

```text
target.self
target.enemy.explicit
target.ally.explicit
target.enemy.first
target.enemy.all
target.ally.all
target.enemy.random
target.enemy.lowest_health
target.ally.lowest_health
target.enemy.adjacent
```

`TargetRequest` 还能按标签过滤并限制最大目标数量。新增长链、扇形、召唤物优先
或仇恨目标时，注册新的选择器即可，不修改 `BattleEngine`。

选择器必须返回存活、已登记且不重复的实体。公共注册表会再次校验自定义选择器
结果，防止错误目标进入 Ability。

每个战斗 Ability 还必须登记 `BattleAbilityTargeting`，明确允许的选择器和最大
目标数。行动请求只能从白名单中选择，玩家不能通过提交 `target.enemy.all` 把
单体技能篡改成群攻。

## 多目标 Ability

一个 Ability 面对多个目标时：

- 资源只支付一次。
- 冷却只设置一次。
- 自身 Effect 只施加一次。
- 目标 Effect 按选择器返回顺序逐个施加。
- 每个目标产生独立伤害、护盾、击败和 Trigger 事实。
- 前一个目标触发的状态能够影响后一个目标。

如果资源消耗公式读取目标属性，使用选择器返回的第一个目标作为主目标。内容
设计应优先让群体能力消耗只依赖使用者属性或显式参数。

## 持续伤害和来源

流血、中毒和灼烧由持续 Effect 授予 Trigger。Trigger 监听
`combat.turn.started`，再用 `DealDamage` 结算。

```text
施法者施加中毒 Effect
→ Effect 保存 source_id
→ 受害者回合开始
→ TriggerSource.GRANT_SOURCE 找回原施法者
→ DealDamage 以原施法者为伤害来源
```

同一种持续伤害可以按来源独立叠层。击败归属、吸血和日志不会错误地记到受害者
自己身上。

## 控制状态

`BattleRules.incapacitating_tags` 定义不能行动的标签。默认支持眩晕、冻结和睡眠
语义标签。行动者回合开始的持续伤害会先结算，再检查控制：

- 已败退：跳过行动。
- 携带禁止行动标签：跳过行动。
- 主动传入空行动：视为放弃或超时。

即使跳过行动，也会执行回合结束和持续时间缩短，因此一回合眩晕能够正常到期。
沉默、缴械等只禁止特定 Ability 的状态，应由 Ability 标签条件判断，不应全部
塞进禁止行动标签。

控制命中、抵抗和韧性后的持续时间由 `ControlEngine` 统一计算。

## 稳定事实

编排层新增：

```text
combat.battle.started
combat.round.started
combat.turn.started
combat.turn.skipped
combat.turn.ended
combat.battle.finished
combat.participant.joined
combat.participant.left
combat.timeline.extra_turn_requested
combat.timeline.delay_requested
combat.action.interrupted
```

它们与伤害层的 `combat.attack.*`、`combat.damage.*`、`combat.shield.*` 事实共同
组成完整战报来源。

## 边界

本层不负责玩家输入、自动战斗 AI、奖励、掉落、体力、数据库事务、QQ 文本或
网页表现。它们以后以 `BattleAction` 输入战斗，以 `BattleStepResult` 和
`RuleEvent` 读取结果。
