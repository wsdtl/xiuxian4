# 随机物品化底座说明

当前版本为 `itemization.foundation.v2`。

武器和装备共用随机属性、档位、滚值、估值和生成凭据。武器策略必须先选择一个唯一核心，并按属性
标签检查后续机制兼容；装备策略不执行流派兼容判断，允许跨方向组合。

生成顺序固定为：

```text
抽取属性数量
抽取属性与档位
滚动显式参数
解析 ContributionSpec
严格价值评估
根据总价值映射品质
保存生成凭据
```

品质是生成结果，不是生成前提。生成策略的品质区间必须连续，最后一档无上限。低价值和高价值结果
都来自同一确定性随机过程，内容包通过属性权重、档位权重和滚值范围控制分布。

生成物品不可在本底座中重铸或合成。交易、兑换、回收和掉落来源属于上层玩法。生成凭据保存内容
指纹、生成器版本、最终属性档位与滚值；持久化恢复时可以重新解析并拒绝内容漂移。

`ItemizationBalanceAuditor` 使用生成器本身、固定种子和固定逻辑时间批量抽检全部策略，不复制估值
公式，也不运行完整战斗。当前七十二条武器策略每条抽取 `128` 次共 `9216` 个实例，通常约四秒完成，
同时输出品质分布、价值区间、均值和平均重试次数。
