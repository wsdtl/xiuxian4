# 交换契约底座说明

## 封板版本

```text
exchange.foundation.v1
```

## 生命周期

```text
开放 -> 买方承诺 -> 成交
  |          |
  +----------+-> 取消或到期
```

开放时，卖方资产使用库存 `escrowed` 预约。部分物资先拆成具有稳定 ID 的独立物资堆，再预约该物资
堆。买方承诺时，账本在付款账户建立资金冻结。成交时，库存转移和资金冻结捕获必须由同一个外层
SQLite 工作单元提交。

## 报价快照

`ExchangeQuote` 保存报价 ID、版本、币种、总额、策略 ID、策略版本和明确分配：

```text
卖方到账
系统或地方费用
其他合法分成
```

所有分配必须使用整数最小货币单位并严格等于总额。税率、参考价和议价规则由游戏次核心计算，
核心只冻结结果并验证账本执行。

## 边界

- 固定价市场使用开放、承诺和成交三步。
- 指定挑战或私下交易通过 `allowed_counterparty_id` 限制对手。
- 竞价活动由活动底座决定赢家，再为赢家创建最终交换契约；交换核心不复制竞价排名。
- 契约到期由周期 worker 显式提交，读取市场列表不会顺手取消契约。

## 不属于本底座

- 商品估值、税率、手续费比例和市场推荐。
- 上架数量、购买资格、价格保护和反小号规则。
- 拍卖竞价、玩家议价和挑战胜负。
- 玩家可见市场、按钮、通知和成交文案。
