# 队伍底座说明

## 封板版本

```text
party.foundation.v1
```

队伍是跨探险、活动和战斗复用的临时成员关系。它只保存队长、成员、站位、准备状态和生命周期，
不保存角色面板、奖励、掉落或具体玩法进度。

## 支持能力

- 创建队伍、加入、离队和踢出成员。
- 转让队长、调整站位和全员准备检查。
- 队长单独离队时自动解散；多人队伍必须先转让队长或显式解散。
- 同一分片内，一个主体只能属于一支活跃队伍。
- 成员或站位变化会清空整队准备状态。
- 所有命令使用 `expected_revision`，失败不改变原状态。

邀请和玩家同意继续使用社会请求底座。持久化联合服务必须在一个 SQLite 事务中完成“接受邀请”和
“加入队伍”，不能先把邀请标记成功再单独尝试入队。

创建社会请求时使用 `party_invitation_metadata(party_id)` 绑定目标队伍，避免业务组件手写元数据键。

## 战斗接入

`PartyBattleProjector` 把所有成员转换成相同 `team_id` 的 `BattleParticipant`，并保留队伍站位。
战斗核心据此自然支持友军治疗、敌方溅射、相邻目标和按阵营判断胜负。

队伍 ID 可以是数据库或业务 ID，不要求符合稳定 ID 格式。投影器使用 SHA-256 派生稳定的战斗阵营
ID，不把玩家可见队名变成规则键。

## 不属于本底座

- 组队经验、掉落倍率和奖励分配。
- 探险、首领或活动的准入条件。
- 匹配算法、跨服房间和断线托管。
- QQ 邀请文案、按钮和命令。
