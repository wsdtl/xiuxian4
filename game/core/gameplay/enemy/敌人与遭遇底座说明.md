# 敌人与遭遇底座说明

## 版本

```text
enemy.foundation.v1
combat-ai.foundation.v2
```

## 固定边界

- 玩家和敌人使用同一套 `Ability -> Effect -> Trigger/Interceptor` 战斗规则。
- 敌人阶位只有普通、精英和首领；世界首领由首领阶位与全服遭遇范围组合。
- 行为模板决定能力、AI、属性倾向、互斥关系和威胁价值，可以持续注册。
- 普通和精英实例由遭遇种子临时生成，不建立独立存档身份。
- 世界首领的共享血量和参与结算属于活动实例，不属于敌人定义。
- 名称、精英前缀、描述和图标全部属于世界皮肤。

## 统一战斗入口

`EnemyCombatProjector` 把等级档案、阶位和行为贡献投影为 `RuleEntity`。敌人 Ability
复用玩家武器 Ability 已登记的 Effect 引用和目标规则，但使用独立稳定 ID，保证战报名称正确。

`BattleAiEngine` 只选择 Ability 与目标模式。它不执行伤害、不修改战斗状态，也不拥有怪物专用
公式。固定种子下，遭遇生成、精英前缀和 AI 同优先级选择都可以重放。

## 遭遇与奖励

`EnemyEncounterGenerator` 根据类型化遭遇定义生成实例，并在生成时排除互斥行为。
`EnemyThreatEvaluator` 使用公共价值引擎和行为威胁加值报价；`EnemyDefeatRewardPlanner`
只汇总经验与掉落表请求，最终资产发放仍交给统一奖励结算。

## 禁止事项

- 禁止为怪物复制伤害、控制、恢复和触发公式。
- 禁止通过玩家可见名称判断敌人类型或行为。
- 禁止让普通敌人拥有玩家库存、武器实例或六槽装备。
- 禁止由命令组件直接随机行为、修改血量或发放掉落。
