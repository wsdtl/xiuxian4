# 异步行动底座

行动系统位于周期、战斗和奖励之间，只回答玩家当前正在做什么，以及行动何时可以形成结果。

## 边界

- `ActionDefinition` 由内容包声明占用主行动、委托行动或即时行动槽，以及持续时间和终止规则。
- `ActionSnapshot` 在开始时冻结规则版本、内容指纹、随机种子和必要角色版本。
- `ActionResult` 只记录已经发生的领域事实和奖励结算 ID，不直接修改角色、库存或账本。
- `ActionState` 只保存进行中和待领取行动。领取、取消和中断后的长期审计由领域事件与事务记录承担。
- 具体探险、休息和首领玩法负责计算结果，但不能自行实现行动槽、计时或领取防重。

## 生命周期

```text
开始 -> 进行中 -> 到期完成 -> 领取后移出当前状态
              -> 主动取消
              -> 外部中断
```

同一角色最多有一个进行中的主行动。委托槽数量由组合根配置；即时行动持续时间必须为零且不占持续槽位。
