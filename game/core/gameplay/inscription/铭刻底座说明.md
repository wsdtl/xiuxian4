# 铭刻底座说明

## 封板版本

```text
inscription.foundation.v1
```

## 一句话理解

铭刻消耗一枚拥有独立标题和故事的铭刻之羽，只修改某件武器、装备或该武器自带 Ability 的玩家
可见名称，不改变任何战斗规则。

## 唯一目标

铭刻只接受三个目标：

```text
武器实例名称
装备实例名称
武器自带 Ability 名称
```

目标使用两个类型化对象表达：

```python
AssetInscriptionTarget(asset_id="weapon-17")
WeaponAbilityInscriptionTarget(
    weapon_asset_id="weapon-17",
    ability_id="ability.weapon_slash",
)
```

资产目标必须是当前玩家拥有的武器或装备独立实例。Ability 目标必须由指定武器在当前定义、品质与
等级状态下真实提供，不能给任意 Ability 写名字。

## 实例数据

铭刻数据保存在目标 `ItemInstance.data["instance_data.inscription"]`：

```text
asset_name
ability_names[ability_id]
```

它不写入 `WeaponState`、`EquipmentState`、物品定义或世界皮肤。这样名称天然跟随具体资产交易，
资产销毁后也天然一起消失，不会产生另一份需要同步清理的铭刻表。

物品实例自身的 revision 会在铭刻后增加。库存快照也按正常库存事务增加 revision，因此旧页面或
并发命令不能覆盖较新的铭刻结果。

## 铭刻之羽

铭刻之羽是带 `item.inscription_medium` 标签的独立物品实例。实例数据必须包含：

```python
InscriptionMediumData(
    title="旧桥挑灯客遗羽",
    flavor_text="灯穗上落下一枚昏黄铭刻之羽……",
)
```

它可以由统一奖励系统通过 `InstanceItemReward.data` 正常发放。铭刻成功后羽毛被消费，成功凭据保留
本次标题和故事供消息展示；目标资产只留下 `custom_name`，不会永久复制羽毛故事。

## 展示规则

基础名必须先由当前 `SkinProjector` 取得，再交给 `InscriptionProjector`：

```text
世界皮肤基础名：青岚短剑
铭刻名：青云剑

默认：青云剑（青岚短剑）
隐藏原名：青云剑
没有铭刻：青岚短剑
```

`InscriptionPreference` 按玩家保存 `show_original_name`。武器装备展示会先把品质和基础名组成完整原名；
开关开启时显示“铭刻名（完整原名）”，关闭时铭刻名直接覆盖完整原名。偏好只影响当前玩家看到的文本，不跟随资产
交易，也不修改铭刻记录。

铭刻名限制为 1 到 12 个字符，禁止空白、控制字符和 Markdown 结构字符。铭刻名不能成为命令、
世界皮肤别名、稳定 ID 或业务查找键。

## 原子事务与防重

一次 `InscriptionEngine.apply()` 按固定顺序执行：

1. 校验库存和目标资产 revision。
2. 校验目标与铭刻之羽都归当前玩家所有。
3. 校验目标是武器、装备或武器真实拥有的 Ability。
4. 校验铭刻之羽标签、标题和故事。
5. 消耗铭刻之羽。
6. 更新目标物品实例铭刻数据。
7. 产生库存消费事实与 `inscription.applied` 事实。

任一步失败都返回原库存。`PersistedInscriptionService` 在同一个 SQLite 工作单元中提交库存快照、
事务防重凭据和 Outbox。相同事务 ID 与相同内容重放只返回原凭据，不会再次消费羽毛；同一事务 ID
对应不同内容会抛出 `TransactionMismatch`。

## 成功消息需要的数据

`InscriptionReceipt` 已包含：

```text
目标
铭刻名
羽毛资产 ID
羽毛标题
羽毛故事
是否为防重重放
```

组件可以据此沿用原有表达：

```text
铭刻名: 青云剑（青岚短剑）
旧桥挑灯客遗羽散作微光, 旧念已入其名.
```

消息文本不进入规则和持久化反推。

## 不属于本底座

- 铭刻之羽的掉落概率和具体首领故事。
- 玩家命令、按钮和 QQ Markdown 排版。
- 世界皮肤中的武器、装备和 Ability 基础名称。
- 任何攻击、防御、属性、Ability 或 Trigger 数值变化。

## 验证

```powershell
.venv\Scripts\python.exe -X utf8 -B test\inscription_foundation_test.py
```

测试覆盖武器名、装备名、武器 Ability 名、原名显示开关、资产交易跟随、非法 Ability、他人资产、
失败原子性、SQLite 重连、Outbox 和重复请求不重复消费。
