# 抽取底座说明

## 封板版本

```text
draw.foundation.v1
```

## 职责

抽取底座只负责把一个可信抽取池解析成随机结果。它复用 `loot.foundation.v1` 的整数权重、批量
抽取、版本号、审计采样和保底状态，不另建一套随机算法。

一个抽取池声明：

- 抽取池自己的稳定 ID 和版本；
- 消耗的签对应的物品 ID；
- 使用的掉落表 ID；
- 允许出现的奖励 ID 集合；奖励可以由玩法层翻译成物品、货币或其他类型化奖励。

启动组装时会检查掉落表中的每一个条目都属于可信奖励集合，并且不能出现空奖。玩家输入不能携带
奖池 ID、奖励 ID 或权重；这些内容只能来自正式内容包。奖池不能把自己的抽取签作为奖励，避免形成
不受来源预算控制的自循环。

## 事务边界

本底座的 `DrawEngine` 只返回 `DrawExecution`，不直接访问数据库、命令或 QQ。需要直接处理堆叠
物品的玩法可以使用 `DrawInventoryEngine`，它仍然只在内存快照上执行，不拥有持久化状态。持久化
玩法必须在同一个工作单元内完成：

1. 校验并消耗抽取签；
2. 执行 `DrawEngine.draw()` 或 `DrawInventoryEngine.execute()`；
3. 把结果翻译为可信的物品、货币或其他类型化奖励；
4. 提交抽取状态和奖励发放。

任一步失败都不能消耗抽取签，也不能留下半份奖励。具体的事务协调放在玩法层，核心不拥有库存
或掉落快照。

## 明确不做

- 不定义世界皮肤下的物品名称；
- 不定义探险、首领或活动掉率；
- 不提供 QQ 命令、按钮和展示；
- 不允许玩家自带奖励 ID、权重或奖池参数；
- 不把抽取签和经济彩票的选号票据混为一谈。
