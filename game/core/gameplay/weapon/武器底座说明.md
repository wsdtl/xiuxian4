# 武器底座说明

## 封板版本

```text
weapon.foundation.v1
```

武器是角色主要战斗循环和长期培养轴。角色可以持有多把武器，但装配底座同时只激活一把。

## 唯一成长结构

```text
武器定义 + 品质 + 等级 + 当前经验
```

- 武器定义决定核心 Ability、Trigger、标签和基础节奏。
- 品质选择对应成长上限、品质贡献和等级数值表。
- 等级决定当前固定数值。
- 武器经验使用独立原子事务增加，可以一次连续升级。

武器不使用隐藏成长公式。每个品质明确提供升级经验表和逐级完整属性值：

```text
experience_requirements = (100, 200)
combat.attack = (20, 35, 50)
```

低品质武器仍然拥有武器定义中的核心玩法；品质调整成长空间与机制强度，不负责第一次开放玩法。

## 角色贡献

`WeaponContributionProvider` 按顺序合并：

```text
武器基础贡献
+ 当前品质贡献
+ 当前等级属性值
= 一份来源为武器实例的 CharacterContribution
```

战斗仍只读取最终 `RuleEntity`，武器组件不实现第二套伤害公式。

## 结构化事实

```text
weapon.experience.gained
weapon.level.increased
```

武器状态使用 `revision` 防止旧快照覆盖。

## 不属于本底座

- 具体武器、品质名称和数值。
- 装备流派。
- 附魔、技能书、宝石和孔位。
- 武器掉落、升级费用和回收价格。
- 玩家可见基础名称由世界皮肤提供，实例铭刻由独立铭刻底座承接。
