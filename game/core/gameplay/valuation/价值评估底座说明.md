# 价值评估底座说明

价值系统为随机属性生成和数值巡检提供快速护栏，不负责替玩家决定最终配装。

属性价值使用显式分段曲线，分别登记属性 ID 与修改层。Ability、Trigger、伤害干预器、目标约束和
具有直接机制价值的标签使用类型化机制价值。协同规则只依赖稳定标签，可以表达完整配装才出现的
额外价值。

`ValuationEngine.evaluate()` 返回攻击、生存、续航、节奏、控制和波动六个维度。总价值不包含波动，
避免高随机性被错误当成稳定强度。`marginal()` 计算加入一件物品前后的价值差，用于发现冷门配装
协同，不反向修改物品生成时的品质。

随机池中的贡献必须全部有价值定义。未登记的属性和机制在严格模式下直接拒绝，不能以零价值混入
生成器。具体权重、标准场景和数值曲线由内容包提供，不属于核心。
