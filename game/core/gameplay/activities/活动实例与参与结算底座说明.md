# 活动实例与参与结算底座说明

## 封板版本

```text
activity.foundation.v1
```

活动底座负责有明确时间窗口的公共生命周期：创建、开放、加入、记录贡献、关闭并冻结排名、完成
结算或取消。首领、虫洞、宗门大会、彩票轮次和赛季事件可以复用这套状态机，但核心不知道这些
具体名称。

活动实例冻结定义 ID 和版本。参与记录保存加入时间、贡献、次数和最后参与时间；容量、单人次数
上限、最低排名贡献和并列规则由版本化定义声明。

关闭活动只冻结稳定排名，不直接发奖励。游戏次核心根据冻结排名和活动规则生成权益，再由权益与
奖励底座兑付。这样重跑排名不会重复发奖，修改展示也不会改变结算资格。

活动只能由显式命令推进。排行榜查询、通知和普通状态读取不能顺手开放或关闭活动；周期 worker
负责在正确时间提交状态命令。

## 不属于本底座

- 首领血量、虫洞层数、宗门贡献来源和彩票号码。
- 排名对应的奖励档位、掉落表和权益活动。
- 活动命令、公告、页面、按钮和运营审批。
- 活动历史榜单的长期投影与保留策略。
