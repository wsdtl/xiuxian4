# 装配底座说明

## 封板版本

```text
loadout.foundation.v1
```

装配底座只负责角色当前激活了哪些资产，以及装卸时库存位置是否共同成功。它不解释武器等级、
装备流派、品质强度或战斗机制。

## 固定槽位

当前游戏固定为一个武器槽和六个装备槽：

```text
loadout_slot.weapon
equipment_slot.head
equipment_slot.body
equipment_slot.hands
equipment_slot.waist
equipment_slot.feet
equipment_slot.accessory
```

稳定 ID 不随世界包变化，头部、护甲等玩家可见名称由世界包提供。

`LoadoutState` 的当前槽位和每个 `LoadoutPreset` 都只保存物品实例 ID。同一个实例不能同时占据多个
槽位。当前激活预设与当前槽位必须完全一致，不能出现展示一套、战斗读取另一套。

## 物品接入

物品定义通过 `item_component.loadout` 声明允许进入的槽位。装配系统只接受独立实例：

```text
武器物品 -> 只能进入唯一武器槽
装备物品 -> 只能进入自己定义的一个装备槽
```

材料和药物没有装配组件，不能进入装配状态。

## 原子装卸

`LoadoutEngine` 支持装备、替换和卸下，并与 `InventoryEngine` 共同返回新快照：

```text
验证角色所有权
+ 验证物品槽位
+ 修改装配状态
+ 移动物品容器
= 一次完整结果
```

背包已满时替换装备使用 `SwapAssetContainers` 同时交换新旧实例位置，不产生“旧装备先入包导致
临时超容”的中间状态。任意验证失败，装配和库存原快照都不改变。

事务携带 `expected_revision`，防止旧装配状态覆盖新状态。

## 多套配装

`SaveLoadoutPreset`、`DeleteLoadoutPreset` 和 `ActivateLoadoutPreset` 允许上层用一次命令管理整套配装。
激活前会完整检查资产所有权、槽位和库存位置。成对替换使用 `SwapAssetContainers`，因此背包满载时
只要最终状态合法仍可切换；任意一件资产失效时整次切换回滚。

手动装备或卸下会退出当前预设，但不会静默覆盖已保存内容。只有显式保存操作才创建或更新预设。

## 贡献汇总

`LoadoutContributionAssembler` 严格按照当前七个槽位读取贡献。背包中未装备的武器和装备不会
进入角色投影。武器和装备组件分别解释自己的状态，再把统一 `CharacterContribution` 交给角色
投影器。

## 结构化事实

```text
loadout.asset.equipped
loadout.asset.replaced
loadout.asset.unequipped
loadout.preset.saved
loadout.preset.deleted
loadout.preset.activated
inventory.item.swapped
```

## 不属于本底座

- 武器品质、等级和经验。
- 装备流派与品质效果。
- 具体武器和装备内容。
- 掉落、交易、回收和展示文案。
- 铭刻、宝石、附魔、孔位和强化。
