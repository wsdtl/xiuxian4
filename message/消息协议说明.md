# 公共消息协议

`message/` 是业务输出与通信驱动器之间的统一语义协议。业务只描述“要表达什么”，驱动器负责“在当前平台如何展示”。

正式业务禁止直接拼接 Markdown 引用符号、QQ 按钮键盘或 OpenAPI 请求字段。

## 依赖方向

```text
业务组件
  -> message.Message
  -> 公共 AdapterReplyManager
  -> 当前通信驱动器
  -> 驱动器渲染器
  -> 平台原生协议
```

`message/` 不依赖 `launch/`、QQ 驱动器或未来修仙业务，因此任何通信协议都可以复用同一消息对象。

## 文档消息

```python
from message import Action, M

reply = (
    M.document()
    .header("角色名 称号 Lv49")
    .inline_section("天机", "灵签已开", icon="system")
    .inline_section("提醒", "灵签中奖", icon="notice")
    .section("纳戒", icon="inventory")
    .row(("数量", 3), ("状态", "可用"))
    .item(1, "生骨丹 x1")
    .note("操作结果将在确认后生效")
    .action(Action("view", "查看", "查看 生骨丹", behavior="send"))
    .build()
)

await manager.send(reply, client_id)
```

## 内容结构

| 构造方法 | 含义 | Markdown 展示规则 |
| --- | --- | --- |
| `header(...)` | 消息主标题 | 粗体标题，不进入引用正文 |
| `inline_section(...)` | 天机、提醒等短信息 | 标题和内容位于一层引用的同一行 |
| `section(...)` | 开始一个正文栏目 | 栏目标题使用一层引用 |
| `line(...)` | 当前栏目的普通正文 | 使用两层引用 |
| `field(label, value)` | 单个字段 | 值自动使用强调样式 |
| `row((label, value), ...)` | 同行字段组 | 分隔符和稳定空白由渲染器生成 |
| `item(index, ...)` | 带稳定编号的条目 | 编号属于正文，不承担按钮功能 |
| `blank()` | 栏目内部空行 | 保持当前栏目的归属范围 |
| `note(...)` | 正文后的附加区 | 与正文自动留空，使用一层引用 |
| `action(...)` | 添加交互动作 | 不进入正文，由驱动器生成控件 |
| `build()` | 冻结消息 | 校验动作 ID 后返回不可变对象 |

`line/field/row/item` 必须先调用 `section`，防止正文失去栏目归属。

业务文本如果手写行首 `>` 或独立 `---` 会立即抛出异常。Markdown 结构只能由渲染器生成。

## 富文本片段

```python
doc = (
    M.document()
    .section("物品详情", icon="item")
    .line("品质: ", M.em("珍稀"))
    .line("说明: ", M.strong("不可交易"))
    .line("资料: ", M.link("查看文档", "https://example.com/docs"))
    .line("操作: ", M.command("查看物品", "查看 生骨丹"))
    .build()
)
```

`M.command(...)` 表达内联命令意图。QQ 渲染器会生成 `mqqapi` 命令链接；不支持该能力的协议会按明确的纯文本规则降级。

## 图标

组件必须传图标分类 key：

```python
M.document().section("纳戒", icon="inventory")
```

图标表位于 `message/icons.py` 的 `SECTION_ICONS`。未知 key 会抛出异常，不会被静默忽略，也不能通过 `icon=` 临时传入任意图标字符。

新增图标时只修改公共图标表，并为对应 key 补测试或真实使用场景。

## 交互动作

```python
from message import Action

actions = (
    Action("confirm", "确认", "确认操作", behavior="callback"),
    Action("run", "执行", "执行操作", behavior="send"),
    Action("fill", "回填", "查看 ", behavior="fill", style="secondary"),
    Action("docs", "文档", "https://example.com", behavior="link"),
)
```

| 字段 | 可用值 | 说明 |
| --- | --- | --- |
| `id` | 稳定非空字符串 | 同一条消息内必须唯一；QQ 回调依赖该字段 |
| `behavior` | `callback` | 平台回调，不自动发送聊天命令 |
| `behavior` | `send` | 自动发送命令 |
| `behavior` | `fill` | 只填入输入框，由用户决定何时发送 |
| `behavior` | `link` | 打开普通链接 |
| `style` | `primary` / `secondary` | 主要或次要视觉样式 |
| `permission` | `everyone` / `admins` / `specified` | 客户端权限提示 |
| `reply` | `True` / `False` | 发送命令时是否引用原消息 |

客户端权限只用于交互提示，不能替代服务端鉴权。确认令牌、归属验证和二次校验必须由业务服务完成。

## 图片消息

```python
reply = M.image(image_bytes, "奖励图片")
await manager.send(reply, client_id)
```

图片可以是驱动器支持的二进制、文件对象或路径。QQ 驱动器会先上传图片，再使用媒体消息发送；业务不接触 `file_info`。

## 显式目标发送

脱离当前命令上下文时必须提供明确目标：

```python
from launch.adapter import SendRequest, manager

await manager.send(
    SendRequest(message=reply, target=reply_target),
    client_id,
)
```

公共管理器不会根据历史消息猜测目标。

## 协议降级

支持 Markdown 的驱动器使用统一 Markdown 渲染器；不支持 Markdown 的驱动器使用纯文本渲染器。两种渲染器读取同一语义结构，业务不编写协议分支。

QQ 原生 payload 只允许出现在 `launch.adapter.qq` 和 QQ 协议探针中。正式业务通过公共 `launch.adapter.manager` 发送时，只接受 `message.Message` 或包含该消息的 `SendRequest`。
